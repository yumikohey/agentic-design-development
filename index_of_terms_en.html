About This Index
The story behind how this index was created—and what it demonstrates about agentic AI capabilities.

Generated by Gemini Pro 2.5

This index was generated by providing Gemini Pro 2.5 with access to all 31 chapter documents via the Workspaces tool. The model used its planning capabilities to batch the document retrievals efficiently, then synthesized the full 100+ entry alphabetical index with chapter locators—a task that would take a human indexer hours to complete manually.

The Prompt Used

do you read [Google Doc URL] and all the related documents in the first page? Please extract all the titles for each chapter and print them. Then process the content of all the individual chapters, and create a detailed index used for publishing, a list of words or phrases ('headings') and associated pointers ('locators') to where useful material relating to that heading can be found. I don't want a sample index. I want the full index, with 100 index terms.

Gemini's Reasoning Process (Excerpt)

Gemini's reasoning steps included: "Beginning Index Creation → Defining Index Extraction Process → Refining the Extraction Strategy → Considering Tool Use → Accessing chapter content (extracting URLs) → Fetching Chapter Content (batching calls) → Initiating URL Retrieval → Developing an Extraction Plan → Crafting the Extraction Strategy." This multi-step reasoning exactly mirrors the Planning + Tool Use + Memory Management pattern composition described in the book's Conclusion chapter.

How to Use This Index
Guidelines for navigating the index effectively.

Chapter References

Each entry is followed by its chapter location. For example: "Prompt Chaining - Chapter 1: Prompt Chaining" means the term appears in Chapter 1. Multiple chapter references mean the term spans several chapters. "Appendix A/B/C" refers to the corresponding appendix sections.

Glossary vs. Index

Terms marked "Glossary" have full definitions available in the book's Glossary section. The Index tells you WHERE a term appears; the Glossary tells you WHAT it means. Terms that appear in both the Index and Glossary are foundational concepts that deserve both location tracking and definitional clarity.

Introduction & Appendix Coverage

The Introduction covers: Agent complexity levels (0–3), the 5 Future Hypotheses, frameworks overview, and what makes an AI system an agent. Appendix A covers advanced prompting; Appendix B covers AI applications (browser use, GUI, visual perception); Appendix C covers frameworks (LangChain, CrewAI, LlamaIndex, MetaGPT, Microsoft AutoGen); Appendices D–G cover AgentSpace, CLI agents, reasoning engines, and coding agents.

Key Chapter Locations
The most important location mappings to keep in mind for quick reference.
  • Prompt Chaining (Ch.1) + Routing (Ch.2) + Parallelization (Ch.3) form the foundational "Core Execution" tier—start here for basic agent construction.
  • Reflection (Ch.4) + Tool Use (Ch.5) are the most practically impactful patterns for improving agent quality and grounding early in development.
  • Planning (Ch.6) + Multi-Agent Collaboration (Ch.7) unlock the next tier of capability—multi-step strategic execution and team-based problem solving.
  • Memory Management (Ch.8) is the connective tissue of all multi-turn and long-running workflows—referenced by almost every other pattern.
  • Knowledge Retrieval/RAG (Ch.14) is the primary solution to hallucination in knowledge-intensive applications—required reading for any production agent.
  • Human-in-the-Loop (Ch.13) + Guardrails/Safety (Ch.18) must be read together for any agent that takes real-world actions or faces adversarial inputs.
  • Evaluation and Monitoring (Ch.19) provides the testing and observability framework needed to validate that all other patterns are working correctly in production.
