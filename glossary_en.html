Why Terminology Matters
Precise terminology is the foundation of effective communication and system design in agentic AI.

The Vocabulary Problem

Agentic AI draws from multiple disciplines: machine learning, software architecture, cognitive science, and distributed systems. Without precise shared vocabulary, teams miscommunicate requirements, engineers implement the wrong patterns, and researchers talk past each other. This glossary establishes the common language used throughout the book.

Overloaded Terms

Many terms in AI have multiple meanings depending on context. "Agent" can mean a reinforcement learning actor, a software agent, or an LLM-based autonomous system. "Memory" can mean GPU memory, context window capacity, or persistent storage for conversation history. This glossary defines each term specifically within the agentic design patterns context.

Rapidly Evolving Language

The field moves quickly. New terms emerge monthly (MCP, A2A, SICA, RLVR). Grounding yourself in the stable foundational vocabulary makes it easier to understand new terminology as it emerges—because new concepts almost always build on established ones.

How to Use This Glossary

Core Terms: A–F
Fundamental concepts that underpin the entire field of agentic AI.
Fundamental Concepts
Prompt
The input, typically in the form of a question, instruction, or statement, that a user provides to an AI model to elicit a response. The quality and structure of the prompt heavily influence the model's output, making prompt engineering a key skill for effectively using AI.
See: Appendix A (Advanced Prompting Techniques) · Glossary
Context Window
The maximum number of tokens an AI model can process at once, including both the input and its generated output. This fixed size is a critical limitation—information outside the window is ignored—while larger windows enable more complex conversations and document analysis.
See: Chapter 1 (Prompt Chaining) · Chapter 8 (Memory Management) · Glossary
In-Context Learning
An AI's ability to learn a new task from examples provided directly in the prompt, without requiring any retraining. This powerful feature allows a single, general-purpose model to be adapted to countless specific tasks on the fly.
See: Appendix A · Glossary
Zero-Shot, One-Shot, & Few-Shot Prompting
Prompting techniques where a model is given zero, one, or a few examples of a task to guide its response. Providing more examples generally helps the model better understand the user's intent and improves accuracy for the specific task.
See: Appendix A · Chapter 9 (Learning and Adaptation) · Glossary
Fine-tuning
The process of adapting a general pre-trained model to a specific task using a smaller, specialized dataset. The most common approach is Supervised Fine-Tuning (SFT), where the model is trained on labeled examples of correct input-output pairs. Instruction Tuning focuses specifically on training the model to better follow user commands.
See: Chapter 9 (Learning and Adaptation) · Glossary
Agent-Specific Terms (A–F)
Agent Trajectory
The complete log of an agent's steps while performing a task—all its thoughts, actions (tool calls), and observations. Analyzing trajectories is a key part of debugging and understanding agent behavior, and is the primary evaluation artifact in Chapter 19.
See: Chapter 19 (Evaluation and Monitoring)
Critique Model
A specialized AI model trained to review, evaluate, and provide feedback on the output of another AI model. It acts as an automated critic, helping to identify errors, improve reasoning, and ensure the final output meets a desired quality standard. Central to the Reflection pattern.
See: Chapter 4 (Reflection) · Glossary
Terms in Context: G–L
Terms relating to how agents interact with data, knowledge, and their environment.
Grounding
The process of connecting a model's outputs to verifiable, real-world information sources to ensure factual accuracy and reduce hallucinations. Often achieved with techniques like RAG to make AI systems more trustworthy.
See: Chapter 14 (Knowledge Retrieval) · Glossary
Hallucination
When an AI model generates information that is factually incorrect, fabricated, or not supported by its input context—stated with high confidence. Hallucination is one of the primary failure modes that Grounding (RAG), Reflection, and Self-Correction patterns aim to mitigate.
See: Chapter 4 (Reflection) · Chapter 14 (RAG)
Human-in-the-Loop (HITL)
A design pattern that integrates human oversight and interaction into an agent's workflow. The agent pauses at critical junctures to ask for human feedback, approval, clarification, or direction—rather than operating fully autonomously.
See: Chapter 13 (Human-in-the-Loop) · Glossary via Index
In-Context Learning (see Core Terms)
Referenced here because in the agent context, it is the mechanism by which agents adapt their behavior from examples within their system prompt or conversation history—without retraining. This is how few-shot examples in system prompts improve agent reliability.
See: Appendix A · Chapter 9
Knowledge Retrieval / RAG (Retrieval-Augmented Generation)
A technique that enhances an LLM by connecting it to an external knowledge source at inference time. The agent embeds a query, retrieves relevant document chunks from a vector database, and includes them in the prompt context. This grounds responses in authoritative, current information.
See: Chapter 14 (Knowledge Retrieval) · Chapter 8 (Memory Management) · Glossary
LangChain / LangGraph
LangChain is a framework providing tools and abstractions to chain together LLMs, prompts, tools, and data sources. LangGraph is its extension for building stateful, graph-based agent workflows where nodes represent agent steps and edges represent conditional transitions between them.
See: Chapters 1–8 (primary framework for code examples) · Appendix C
Technical Terms: M–R
Technical concepts covering model architectures, training methods, and agentic frameworks.
Model Architectures
Transformer
The foundational neural network architecture for most modern LLMs. Its key innovation is the self-attention mechanism, which efficiently processes long sequences of text and captures complex relationships between words. Nearly all state-of-the-art language models (GPT, Gemini, Claude, Llama) are Transformer-based.
See: Glossary · Appendix F (Under the Hood)
Mixture of Experts (MoE)
An efficient model architecture where a "router" network dynamically selects a small subset of "expert" networks to handle any given input. This allows models to have a massive number of parameters while keeping computational costs manageable—enabling scale without proportional cost increase.
See: Glossary · Appendix F
Mamba (Selective State Space Model)
A recent AI architecture using a Selective State Space Model (SSM) to process sequences with high efficiency, especially for very long contexts. Its selective mechanism focuses on relevant information while filtering out noise, making it a potential alternative to Transformer-based models for certain tasks.
See: Glossary
Training & Fine-tuning Techniques
PEFT (Parameter-Efficient Fine-Tuning)
Methods for adapting a pre-trained model to a specific task while updating only a small fraction of its parameters. This dramatically reduces computational cost compared to full fine-tuning. Key PEFT techniques include LoRA (Low-Rank Adaptation), QLoRA (memory-efficient LoRA), and prefix tuning.
See: Glossary · Chapter 9 (Learning and Adaptation)
RLHF (Reinforcement Learning from Human Feedback)
The most prominent alignment technique. A "reward model" trained on human preferences guides the AI's learning process, often using Proximal Policy Optimization (PPO). Simpler alternatives include DPO (Direct Preference Optimization, bypasses separate reward model) and KTO (Kahneman-Tversky Optimization, simplifies data collection).
See: Glossary · Chapter 9 (Learning and Adaptation)
Agentic Frameworks & Protocols
Model Context Protocol (MCP)
An emerging standard (Anthropic) for how agents and tools exchange information. MCP defines standardized tool discovery, invocation, and response formats—enabling agents to use tools they were not explicitly programmed for and supporting the "agentic ecosystem" vision.
See: Chapter 10 (Model Context Protocol) · Index
ReAct (Reason and Act)
An agent framework combining reasoning and acting in a loop: Thought (reason about what to do) → Action (use a tool) → Observation (see the result) → repeat until done. ReAct is foundational to the Tool Use and Planning patterns throughout the book.
See: Chapter 5 (Tool Use) · Chapter 17 (Reasoning Techniques) · Appendix A · Glossary
Advanced Terms: S–Z
Advanced concepts for production-grade agentic systems and cutting-edge research areas.
Self-Consistency
A prompting technique where the model generates multiple independent reasoning paths for the same problem and selects the most consistent answer via majority vote. Improves accuracy on complex reasoning tasks by leveraging the diversity of the model's outputs.
See: Appendix A · Chapter 17 (Reasoning Techniques)
SICA (Self-Improving Coding Agent)
An agent architecture where the agent autonomously writes, executes, and evaluates code, using the results to iteratively improve its own codebase or the code it generates. Represents the Learning and Adaptation pattern applied to software development.
See: Chapter 9 (Learning and Adaptation)
Tree of Thoughts (ToT)
An advanced reasoning framework where an agent explores multiple reasoning paths simultaneously, like branches on a tree. The agent self-evaluates different lines of thought and chooses the most promising one to pursue, making it more effective at complex problem-solving than linear chain-of-thought.
See: Chapter 17 (Reasoning Techniques) · Appendix A · Glossary
Chain of Thought (CoT)
A prompting technique that encourages a model to explain its reasoning step-by-step before giving a final answer. This "thinking out loud" process often leads to more accurate results on complex reasoning tasks by making intermediate reasoning explicit and checkable.
See: Chapter 17 (Reasoning Techniques) · Appendix A · Glossary
Deep Research
An agent's capability to autonomously explore a topic in-depth by iteratively searching for information, synthesizing findings, and identifying new questions to investigate. Allows the agent to build comprehensive understanding far beyond what a single search query provides.
See: Chapter 6 (Planning) · Chapter 17 (Reasoning Techniques) · Glossary
RLVR (Reinforcement Learning with Verifiable Rewards)
A training approach where the reward signal comes from verifiable, ground-truth feedback rather than human preferences. Particularly powerful for mathematical reasoning, coding, and other domains where correctness can be automatically verified—enabling scaling laws in agent training.
See: Chapter 17 (Reasoning Techniques)

Key Definitions to Remember
The most important terms that appear repeatedly throughout the book.
  • Context Window — The hard limit on how much an LLM can "see" at once. Everything outside it is invisible to the model. This single constraint drives the design of Memory Management and RAG patterns.
  • Grounding — Connecting AI outputs to verifiable facts. Without grounding, agents hallucinate. RAG, Tool Use, and Reflection all contribute to grounding in different ways.
  • ReAct (Reason and Act) — The foundational loop of modern agents: Think → Act → Observe → repeat. Most Tool Use and Planning implementations follow this pattern explicitly.
  • RLHF — How most production LLMs are aligned with human values. Understanding this explains why models refuse certain requests, have certain default behaviors, and how fine-tuning for specific agent roles works.
  • Trajectory — The audit trail of everything an agent does. Critical for debugging, evaluation, and building trust in autonomous systems. Always log trajectories in production agents.
  • MCP (Model Context Protocol) — The emerging standard for how agents discover and use tools. Understanding MCP is essential for building agents that can integrate with an expanding ecosystem of services.
