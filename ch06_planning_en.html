Planning

The pattern that elevates agents from reactive responders to strategic executors — autonomously decomposing complex goals into coherent, actionable sequences of steps, adapting when circumstances change.

What is the Planning Pattern?

Intelligent behavior often involves more than just reacting to immediate input. It requires foresight, breaking down complex tasks into smaller manageable steps, and strategizing how to achieve a desired outcome. At its core, planning is the ability for an agent or system of agents to formulate a sequence of actions to move from an initial state towards a goal state. The plan is not known in advance; it is created in response to the request.

Adaptability Is the Hallmark

A hallmark of the Planning pattern is adaptability. An initial plan is merely a starting point, not a rigid script. The agent's real power is its ability to incorporate new information and steer the project around obstacles. A capable agent doesn't simply fail when a preferred option becomes unavailable — it adapts, re-evaluates its options, and formulates a new plan.

The Critical Trade-off: When NOT to Plan

Dynamic planning is a specific tool, not a universal solution. When a problem's solution is already well-understood and repeatable, constraining the agent to a predetermined, fixed workflow is more effective. This limits the agent's autonomy to reduce uncertainty and guarantees reliable, consistent outcomes. The key question: does the "how" need to be discovered, or is it already known?

Real-World Example: Google DeepResearch

Google Gemini DeepResearch is a concrete example of the Planning pattern in production. It begins by deconstructing a user's prompt into a multi-point research plan (presented for user review), then initiates an iterative search-and-analysis loop. The agent dynamically formulates and refines queries based on gathered information, identifying knowledge gaps and corroborating data points — classic adaptive planning behavior.

Problems Planning Solves
Complex problems cannot be solved with a single action and require foresight to achieve a desired outcome. Click each card to explore the specific failures this pattern addresses:

Cannot Handle Multi-Step Tasks

Reactive agents fail on tasks requiring sequences of interdependent actions

A simple reactive agent can respond to a single input but cannot manage a task like "onboard a new employee" that requires creating accounts, assigning modules, and coordinating with departments in a logical order. Planning provides the framework for orchestrating complex sequences.

No Goal-Oriented Behavior

Agents without planning lack the ability to proactively work toward complex objectives

Without planning, agents simply react to inputs. They cannot autonomously pursue a goal across multiple steps, tool calls, and decision points. Planning transforms an agent from a passive responder into a strategic executor that proactively works toward achieving objectives.

Fails on Unexpected Obstacles

Rigid workflows fail when circumstances change mid-execution

A hardcoded workflow fails when a critical resource becomes unavailable or a step produces unexpected results. Planning agents can re-evaluate their options, adapt their approach, and formulate a new course of action when the original plan encounters obstacles.

Cannot Manage Dependencies

Complex tasks have interdependent steps that must execute in the right order

Many real-world tasks involve dependencies — step B must happen after step A, certain information must be gathered before decisions are made, some tools can only be called with outputs from previous steps. Planning provides the logical framework to identify and manage these dependencies correctly.

How Planning Works
The Planning pattern structures complex goal achievement into distinct phases. Click each step to understand the mechanics:
1
Goal Understanding
Agent understands initial state and goal state
The agent first understands the initial state (current conditions, available resources, constraints) and the goal state (what "done" looks like). It defines the "what" — the objective and its constraints — but not yet the "how." This context is essential for generating a relevant and realistic plan.

Example: Goal: "Organize a team offsite" → Constraints: budget, participant count, desired dates → Success: confirmed bookings
2
Plan Generation
LLM decomposes goal into actionable sub-steps
The LLM decomposes the high-level objective into a structured sequence of smaller, actionable steps or sub-goals. LLMs are particularly well-suited for this, as they can generate plausible and effective plans based on their vast training data. The plan may be presented to the user for review before execution (as in DeepResearch).

Output: Ordered list of concrete sub-tasks with dependencies identified
3
Plan Execution
Agent executes steps, invoking tools as needed
The agent follows the plan sequentially, executing each sub-task. This typically involves invoking tools, querying databases, calling APIs, or interacting with other agents. The execution phase transforms the abstract plan into concrete actions with real-world effects. State is maintained across steps to enable dependencies.

Example (DeepResearch): Iteratively executes web searches, analyzes results, identifies knowledge gaps, refines queries
4
Adaptive Re-planning
Agent adjusts plan based on new information
When execution reveals unexpected results, resource unavailability, or new information, the agent re-evaluates the current state and updates the plan. This adaptability distinguishes a true planning agent from a simple fixed workflow. The agent registers the new constraint, re-evaluates its options, and formulates an alternative course of action.

Example: Preferred venue unavailable → agent searches alternatives → updates plan with new venue
Core Concepts

Goal Decomposition

The core cognitive operation of planning is decomposing a high-level objective into a directed sequence of sub-tasks. Each sub-task is smaller, more concrete, and manageable. The decomposition must account for dependencies (which steps must come before others), resource requirements, and potential failure points. LLMs can perform this decomposition naturally from task descriptions.

State Tracking Across Steps

Effective planning requires maintaining state across the execution of multiple steps — tracking what has been accomplished, what resources have been consumed, what results were obtained, and what dependencies have been satisfied. Frameworks like CrewAI and LangGraph provide built-in state management for this purpose.

Planning vs. Fixed Workflow

The fundamental question is: does the "how" need to be discovered, or is it already known? If the solution is well-understood and repeatable, a fixed workflow is more reliable, predictable, and cost-effective. Dynamic planning is appropriate when the path to the goal cannot be predetermined — due to unknown information, variable environments, or complex dependencies that can only be resolved during execution.

Practical Applications
The Planning pattern enables agents to move beyond simple reactive actions to goal-oriented behavior. Select a use case:

Autonomous Deep Research (Google DeepResearch)

Multi-step research that iteratively queries, analyzes, and synthesizes information from web sources.

1
Goal: Produce a comprehensive, cited research report on a complex topic.
2
Planning: Decomposes the topic into a multi-point research plan: search for overview → identify key players → focus on growth data → look for sector analysis.
3
Execution: Iteratively searches Google, evaluates sources, identifies knowledge gaps, refines queries, synthesizes findings into a structured report with citations.

Procedural Task Automation (Employee Onboarding)

Orchestrating complex workflows with multiple steps, dependencies, and system integrations.

1
Goal: Onboard a new employee — create accounts, assign training, coordinate departments.
2
Planning: Decomposes into: create email account → set up HRIS profile → assign training modules → notify IT → schedule orientation → assign buddy.
3
Execution: Agent executes steps in logical order, invoking APIs for each system, managing dependencies (email must exist before HRIS profile is linked).

Robotics and Autonomous Navigation

State-space traversal — finding a path from initial state to goal state while avoiding obstacles.

1
Goal: Navigate a physical or virtual environment from point A to point B.
2
Planning: Generates a path optimized for time, energy, or safety, while accounting for obstacles and environmental constraints.
3
Adaptive: Detects unexpected obstacles at runtime and re-plans an alternative route to reach the goal.

Structured Information Synthesis (Research Reports)

Generating complex outputs by planning distinct phases: research → structure → draft → refine.

1
Goal: Write a comprehensive research report or technical document.
2
Planning: Formulates a plan with distinct phases: information gathering → data summarization → content structuring → iterative refinement.
3
Execution: Agent follows the plan, using tools at each phase — search tools for gathering, writing tools for drafting, evaluation tools for refinement.

Multi-Step Customer Support Resolution

Systematic diagnosis and resolution following a planned approach with escalation paths.

1
Goal: Resolve a complex customer issue that may require multiple investigation and action steps.
2
Planning: Creates a systematic plan: diagnose issue category → check account status → search known issues → apply fix → verify resolution → escalate if needed.
3
Adaptive: If standard fixes don't work, re-plans by escalating to a human agent or trying alternative resolution paths.

Event Planning and Coordination

Complex coordination tasks with many interdependent components and external dependencies.

1
Goal: Organize a team offsite meeting with venue, catering, accommodation, and logistics.
2
Planning: Decomposes into: search venues → check availability → get catering quotes → book accommodation → arrange transport → send invites.
3
Adaptive: If preferred venue is unavailable, adapts by searching alternatives; if dates conflict, re-plans with adjusted timeline.
Code Examples
Planning pattern implementation in CrewAI (plan-then-write agent) and OpenAI Deep Research API (autonomous research planning).

CrewAI: Planner-Writer Agent

from crewai import Agent, Task, Crew, Process from langchain_openai import ChatOpenAI from dotenv import load_dotenv load_dotenv() llm = ChatOpenAI(model="gpt-4-turbo") # --- Define a Planner-Writer Agent --- planner_writer_agent = Agent( role='Article Planner and Writer', goal='Plan and then write a concise, engaging summary on a specified topic.', backstory=( 'You are an expert technical writer and content strategist. ' 'Your strength lies in creating a clear, actionable plan before writing, ' 'ensuring the final summary is both informative and easy to digest.' ), verbose=True, llm=llm ) # --- Define Task with explicit planning step --- topic = "The importance of Reinforcement Learning in AI" high_level_task = Task( description=( f"1. Create a bullet-point plan for a summary on: '{topic}'.\n" f"2. Write the summary based on your plan, keeping it around 200 words." ), expected_output=( "A final report with two distinct sections:\n" "### Plan\n- A bulleted list outlining the main points.\n\n" "### Summary\n- A concise, well-structured summary." ), agent=planner_writer_agent, ) # --- Assemble and run the Crew --- crew = Crew( agents=[planner_writer_agent], tasks=[high_level_task], process=Process.sequential, ) result = crew.kickoff() print(result)

OpenAI Deep Research API

from openai import OpenAI client = OpenAI(api_key="YOUR_OPENAI_API_KEY") # Define agent role and research question system_message = """You are a professional researcher preparing a structured, data-driven report. Focus on data-rich insights, use reliable sources, and include inline citations.""" user_query = "Research the economic impact of semaglutide on global healthcare systems." # The model autonomously creates a research plan, searches the web, # analyzes sources, and synthesizes a cited report response = client.responses.create( model="o3-deep-research-2025-06-26", input=[ {"role": "developer", "content": [{"type": "input_text", "text": system_message}]}, {"role": "user", "content": [{"type": "input_text", "text": user_query}]} ], reasoning={"summary": "auto"}, tools=[{"type": "web_search_preview"}] ) # Access final report and inspect intermediate steps final_report = response.output[-1].content[0].text print(final_report)
Deep Dive: Planning in Practice
Google DeepResearch: Planning at Scale
An end-to-end example of the Planning pattern in production
Google Gemini DeepResearch exemplifies advanced Planning in production. It begins by deconstructing a prompt into a multi-point research plan presented to the user for review and modification (enabling collaborative plan refinement). Once approved, it initiates an iterative search-and-analysis loop, dynamically formulating and refining queries based on gathered information, identifying knowledge gaps and corroborating data points. The final output is a structured, multi-page report with citations. The entire asynchronous process is resilient to single-point failures, allowing the user to disengage and be notified upon completion.
OpenAI Deep Research API Transparency
The API exposes intermediate reasoning, searches, and code execution
Unlike abstracted systems like ChatGPT, the OpenAI Deep Research API exposes all intermediate steps — the agent's internal reasoning steps, the specific web search queries it executed, and any code it ran. This transparency enables detailed debugging and a deeper understanding of how the final answer was constructed. It also supports the Model Context Protocol (MCP), enabling developers to connect the agent to private knowledge bases and internal data sources.
Dynamic vs. Static Planning
Choosing between adaptive planning and fixed workflows
Static (fixed) workflows: optimal when the solution path is known, repeatable, and needs to be reliable. Use for well-understood processes like order fulfillment or data transformation. Dynamic planning: optimal when the path to the goal cannot be predetermined — novel research topics, complex scheduling with many constraints, or tasks where requirements evolve during execution. The Planning pattern adds power but also complexity and unpredictability. Default to fixed workflows unless the task genuinely requires discovering the "how" at runtime.
Prompting for Planning Behavior
How to elicit planning behavior from LLMs in agent frameworks
Explicitly prompting or designing tasks to require planning steps encourages this behavior. In CrewAI, the task description explicitly instructs the agent to first create a plan, then execute it ("1. Create a plan... 2. Write the summary based on your plan"). In LangGraph, planning can be structured as a distinct node. The agent's backstory and goal definition also shape planning behavior — emphasizing "you create a clear, actionable plan before executing" in the backstory reliably induces planning-first behavior.
Integration with Other Patterns
Planning works best when combined with Tool Use, Reflection, and Memory
Planning is most powerful when combined with other patterns. Tool Use enables the agent to execute plan steps by calling external APIs and services. Reflection allows the agent to evaluate intermediate results and adapt the plan. Memory enables the agent to build on previous research and avoid re-doing work. Together, these patterns create a sophisticated system capable of sustained, goal-directed work on complex real-world problems — as exemplified by DeepResearch, which combines planning, tool use (Google Search), and synthesis.
At a Glance

WHAT

Complex problems cannot be solved with a single action and require foresight. Without a structured approach, agentic systems struggle to break down high-level objectives into manageable sequences, leading to incomplete or incorrect results when faced with intricate goals.

WHY

Planning offers a standardized solution: the agent first creates a coherent plan by decomposing the objective into actionable steps, then executes it. This allows managing complex workflows, orchestrating tools, and handling dependencies — transforming a reactive agent into a strategic executor.

WHEN TO USE

When a user's request is too complex to be handled by a single action or tool. Ideal for multi-step processes like research reports, employee onboarding, competitive analysis, or any task requiring a sequence of interdependent operations to reach a final synthesized outcome.

RULE OF THUMB

Ask: does the "how" need to be discovered, or is it already known? If known → use a fixed workflow. If it must be discovered (novel research, dynamic environments, complex constraints) → use Planning. Dynamic planning adds power but also complexity and unpredictability.

Key Takeaways
  • Planning enables agents to break down complex goals into actionable, sequential steps — transforming reactive responders into strategic, goal-oriented executors.
  • The hallmark of the Planning pattern is adaptability: an initial plan is a starting point, not a rigid script. The agent can incorporate new information, handle obstacles, and re-formulate its approach mid-execution.
  • The key architectural question is: does the "how" need to be discovered at runtime, or is it already known? If known, a fixed workflow is more reliable; Planning is for tasks where the path must be discovered.
  • LLMs are well-suited for planning tasks — they can generate plausible, effective decompositions of complex goals based on their vast training data and natural language understanding.
  • Explicitly prompting planning behavior (e.g., "First create a plan, then execute it") reliably induces planning-first behavior in agent frameworks like CrewAI and LangGraph.
  • Google DeepResearch exemplifies production-scale Planning: it deconstructs a prompt into a research plan, iteratively executes web searches, identifies knowledge gaps, and synthesizes a cited report — all adaptively.
  • The OpenAI Deep Research API provides programmatic access to deep research planning with full transparency: it exposes reasoning steps, search queries, and code execution for debugging and auditability.
  • Planning is most powerful when combined with Tool Use (for executing plan steps), Reflection (for evaluating and adapting), and Memory (for building on previous work).
