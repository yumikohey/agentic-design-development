What is Evaluation and Monitoring?

This chapter examines methodologies that allow intelligent agents to systematically assess their performance, monitor progress toward goals, and detect operational anomalies. While goal setting focuses on what an agent should achieve, evaluation and monitoring focus on how well it is achieving those goals — continuously and externally measuring effectiveness, efficiency, and compliance with requirements.

Why Continuous Evaluation Matters

Agentic systems and LLMs operate in complex, dynamic environments where performance can degrade over time. Their probabilistic and non-deterministic nature means traditional software testing — which yields predictable pass/fail results — is insufficient. Continuous assessment measures an agent's effectiveness, efficiency, and adherence to operational and safety requirements.

Three Pillars of Evaluation

  • Defining clear metrics — accuracy, latency, token usage, helpfulness
  • Establishing feedback loops — continuous measurement and A/B testing
  • Implementing reporting systems — ensuring performance aligns with expectations

From Agents to Contractors

A significant evolution covered in this chapter is the shift from simple prompt-driven agents to advanced "contractors" — agents bound by formal, verifiable agreements. This transformation makes AI reliable enough to deploy in mission-critical, high-stakes environments by embedding principles of negotiation, formal specification, and self-validation into the agent's core logic.

Why Standard Testing Fails for Agents
Traditional software testing is insufficient for agentic AI. Click each failure mode to understand why:

Probabilistic Behavior

Agents don't produce deterministic outputs

Standard code either passes or fails a test. Agents produce probabilistic outputs — the same input can yield different, yet equally valid, responses. This demands qualitative assessment alongside binary pass/fail checks, requiring a fundamentally different evaluation philosophy.

Performance Drift

Agent quality degrades over time

Data drift, environmental shifts, and concept drift can degrade an agent's performance after deployment. An agent performing well at launch may deteriorate as the world changes around it, making continuous monitoring — not just pre-deployment testing — essential.

Trajectory Complexity

Multi-step paths are hard to validate

Agents take sequences of steps (trajectories) to reach a goal. The final output may be correct while the reasoning process was flawed — or vice versa. Both the final output and the full sequence of intermediate actions must be evaluated, which standard tests cannot easily do.

Multi-Agent Complexity

Teamwork is harder to measure than solo work

Evaluating multi-agent systems requires measuring not just individual agent performance but also inter-agent cooperation, handoff quality, and overall system effectiveness. A hotel agent booking a room for the wrong week — due to a failure in flight agent communication — is a system-level problem invisible to individual unit tests.

Subjective Qualities

Helpfulness resists objective measurement

Qualities like "helpfulness," "coherence," and "appropriateness" cannot be measured with simple metrics. A response that is factually correct may still be unhelpful in context. Evaluating these subjective dimensions requires advanced techniques like LLM-as-a-Judge approaches.

The Cost of Poor Evaluation

Without robust evaluation, agents deployed in production can silently degrade, produce harmful outputs, violate compliance requirements, or make costly errors in high-stakes domains. The challenge intensifies in multi-agent systems where failures compound across handoffs and the root cause of a final error may be buried deep in an earlier agent's decision.

A Layered Evaluation Framework
Comprehensive agent evaluation requires multiple complementary approaches. Click each layer to explore:
1
Response Accuracy Assessment
Measure: Factual correctness, fluency, semantic similarity
This core process evaluates whether the agent delivers pertinent, correct, logical, unbiased, and accurate information. Basic exact-match comparison is a starting point, but real-world scenarios require sophisticated metrics: String Similarity (Levenshtein, Jaccard), Keyword Analysis, Semantic Similarity via cosine distance on embeddings, LLM-as-a-Judge evaluations, and RAG-specific metrics like faithfulness and relevance.

Key insight: A simple string comparison between "The capital of France is Paris" and "Paris is the capital of France" will incorrectly return 0.0 accuracy — despite identical meaning — illustrating why semantic evaluation is essential.
2
Latency and Resource Monitoring
Measure: Response time, token counts, operational cost
Speed is a critical factor in real-time applications. Latency monitoring measures the duration required for an agent to process requests and generate outputs. Elevated latency adversely affects user experience and overall agent effectiveness.

Token usage tracking is equally crucial — billing for LLM interactions depends on token counts, so monitoring consumption directly controls costs. Logging should persist to structured stores (JSON, InfluxDB, Prometheus, Datadog) rather than console output alone.
3
Trajectory Evaluation
Measure: Action sequences, tool use, reasoning path
Agents' trajectories — the sequence of steps taken to reach a solution — are essential to evaluate. The agent's actual actions are compared to an expected ground-truth trajectory. Comparison methods include: exact match (perfect sequence), in-order match (correct actions in order, extra steps allowed), any-order match, precision (relevance of predicted actions), and recall (completeness of essential actions).

High-stakes scenarios demand exact match; flexible situations may use in-order or any-order matching.
4
LLM-as-a-Judge
Measure: Helpfulness, coherence, appropriateness
For subjective qualities, a secondary LLM acts as an evaluator, assessing another agent's output based on predefined rubrics. This approach offers nuanced, human-like evaluation that surpasses simple keyword matching.

The judge is prompted with structured criteria (clarity, neutrality, relevance, completeness, appropriateness) and returns structured JSON feedback including overall score, rationale, detailed feedback per criterion, concerns, and recommended action.
Core Concepts

Evaluation Methods Compared

Three primary evaluation approaches each have distinct trade-offs:

  • Human Evaluation — Captures subtle behavior; difficult to scale, expensive, time-consuming
  • LLM-as-a-Judge — Consistent, efficient, scalable; may overlook intermediate steps, limited by LLM capabilities
  • Automated Metrics — Scalable, efficient, objective; may fail to capture complete agent capabilities

Test Files vs. Evalset Files (Google ADK)

Test files (JSON) represent single, simple agent-model interactions — ideal for unit testing during development. Each file contains a session with user query, expected tool use trajectory, intermediate responses, and final response.

Evalset files utilize a dataset ("evalset") for evaluating complex, multi-turn conversations — suited for integration tests. Each eval contains one or more sessions with multiple turns, expected tool use, intermediate responses, and a reference final response.

The Agent Contractor Model

An evolution from simple agents to "contractors" bound by formal agreements. Four pillars make this work:

  • Formalized Contract — Explicit deliverables, data sources, scope, cost, and completion time
  • Negotiation Lifecycle — Agent can flag ambiguities and negotiate terms before execution
  • Quality-Focused Execution — Self-validation loop: generate, review, improve until specs are met
  • Hierarchical Decomposition — Main contract broken into formal subcontracts for specialized agents

Practical Applications
Select a use case to explore how evaluation and monitoring applies in practice:

Performance Tracking in Live Systems

Continuously monitoring the accuracy, latency, and resource consumption of an agent deployed in a production environment.

1
Instrument agent to emit metrics on every request (response time, token count, tool calls made)
2
Log to time-series database (InfluxDB, Prometheus) or observability platform (Datadog, Grafana)
3
Set thresholds and alert when latency exceeds SLA or accuracy drops below baseline
4
Example: customer service chatbot's resolution rate and mean response time tracked hourly

A/B Testing for Agent Improvements

Systematically comparing the performance of different agent versions or strategies in parallel to identify optimal approaches.

1
Deploy two variants — e.g., two different planning algorithms for a logistics agent
2
Route traffic randomly or by cohort to each variant, collecting identical metrics
3
Apply statistical significance tests to determine which variant performs better
4
Gradually shift traffic to winning variant while monitoring for regressions

Compliance and Safety Audits

Generating automated audit reports that track an agent's compliance with ethical guidelines, regulatory requirements, and safety protocols over time.

1
Define compliance criteria as machine-readable rules (output format, forbidden content, required disclosures)
2
Run agent outputs through automated compliance checker on every interaction
3
Generate KPI reports and trigger alerts when violations are detected
4
Route flagged cases to human-in-the-loop review for final adjudication

Drift Detection

Monitoring the relevance or accuracy of an agent's outputs over time, detecting when performance degrades due to changes in input data distribution (concept drift) or environmental shifts.

1
Establish performance baseline at deployment using a held-out reference dataset
2
Periodically re-evaluate agent on the same reference set to detect score changes
3
Monitor input distribution statistics to detect when real-world data diverges from training distribution
4
Trigger retraining or model update pipeline when drift exceeds threshold

Anomaly Detection in Agent Behavior

Identifying unusual or unexpected actions taken by an agent that might indicate an error, a malicious attack, or an emergent undesired behavior.

1
Profile normal agent behavior patterns (typical tool sequences, response length distributions, latency ranges)
2
Deploy statistical or ML-based anomaly detector monitoring agent action logs in real time
3
Alert when agent takes actions outside normal operating envelope (e.g., calling an unexpected API, producing unusually long outputs)
4
Automatically quarantine or roll back agent when severe anomalies are detected

Learning Progress Assessment

For agents designed to learn, tracking their learning curve, improvement in specific skills, or generalization capabilities over different tasks or datasets.

1
Define skill-specific benchmarks at multiple difficulty levels for each target capability
2
Run agent on benchmark suite after each training epoch or fine-tuning cycle
3
Track learning curves across skills — flag catastrophic forgetting or uneven skill development
4
Compare generalization on held-out tasks vs. training tasks to detect overfitting

Multi-Agent System Evaluation

Evaluating complex systems where multiple agents cooperate, requiring assessment of both individual and collective performance.

1
Evaluate cooperation: after a Flight-Booking Agent secures a flight, does it pass correct dates to the Hotel-Booking Agent?
2
Check plan adherence: does the system book flight before hotel, as planned — or does it deviate?
3
Verify agent selection: is the right specialized agent chosen for each sub-task?
4
Test scalability: does adding a new agent (e.g., Restaurant Reservation) improve or degrade overall performance?
Code Examples

Response Accuracy Evaluator

A simple baseline evaluator using exact string matching — illustrating why more sophisticated approaches are needed:

def evaluate_response_accuracy(agent_output: str, expected_output: str) -> float: """Calculates a simple accuracy score for agent responses.""" # Basic exact match — real-world needs more sophisticated metrics return 1.0 if agent_output.strip().lower() == expected_output.strip().lower() else 0.0 # Example — demonstrates the semantic equivalence problem agent_response = "The capital of France is Paris." ground_truth = "Paris is the capital of France." score = evaluate_response_accuracy(agent_response, ground_truth) # Returns 0.0 despite semantically identical meaning! print(f"Response accuracy: {score}") # Output: 0.0

LLM-as-a-Judge for Helpfulness

Using Gemini to evaluate survey question quality across five criteria with structured JSON output:

import google.generativeai as genai, json, os RUBRIC = """You are an expert reviewer. Score the input on 5 criteria (1-5 each): 1. Clarity & Precision 2. Neutrality & Bias 3. Relevance & Focus 4. Completeness 5. Appropriateness for Audience Return JSON with: overall_score, rationale, detailed_feedback, concerns, recommended_action.""" class LLMJudgeForLegalSurvey: def __init__(self, model_name='gemini-1.5-flash-latest', temperature=0.2): self.model = genai.GenerativeModel(model_name) self.temperature = temperature def judge_survey_question(self, survey_question: str): full_prompt = f"{RUBRIC}\n\nQUESTION:\n{survey_question}" response = self.model.generate_content( full_prompt, generation_config=genai.types.GenerationConfig( temperature=self.temperature, response_mime_type="application/json" ) ) return json.loads(response.text)
Deep Dive: Advanced Evaluation Concepts
Click each layer to explore advanced evaluation concepts in depth:
Trajectory Matching Methods
Exact, in-order, any-order, precision, recall
Exact match requires a perfect sequence of actions — used in high-stakes scenarios where any deviation is unacceptable. In-order match allows extra steps as long as required actions appear in order. Any-order match allows correct actions in any sequence. Precision measures the fraction of predicted actions that are actually necessary. Recall measures the fraction of necessary actions that were captured. Choose based on risk: a medical diagnosis agent might need exact match, while a trip planner can tolerate any-order.
Google ADK Evaluation Methods
Web UI, pytest integration, CLI
Google's Agent Development Kit provides three evaluation pathways. The web-based UI (adk web) enables interactive session creation and saving into eval sets — ideal for exploratory testing. Pytest integration allows test files to run as part of CI/CD pipelines by calling AgentEvaluator.evaluate(). The command-line interface (adk eval) facilitates automated batch evaluation for regular build verification. All three methods share the same underlying test/evalset file format.
The Agent Contractor Model in Detail
Formalized contracts for verifiable AI execution
The contractor model (proposed in Agent Companion, Gulli et al.) transforms AI from probabilistic assistants to deterministic contractors. A contract for a financial analysis task doesn't just say "analyze Q1 sales" — it specifies "a 20-page PDF analyzing European market sales from Q1 2025, including five data visualizations, comparative analysis against Q1 2024, and a risk assessment." The agent negotiates unclear terms before starting, self-validates during execution, and only delivers when all contract specifications are met.
Multi-Agent System Evaluation Challenges
Cooperation, handoffs, scalability
Multi-agent evaluation requires a fundamentally different approach because there are many steps and handoffs. Key questions: Are agents cooperating effectively? Are they sticking to the plan? Is the right agent being chosen for each task? Does adding agents improve the system? Each handoff is an opportunity for failure — a Flight-Booking Agent securing a correct flight means nothing if it passes wrong dates to the Hotel-Booking Agent, resulting in the wrong week being booked.
From Prompts to Formal Specifications
Why informal instructions fail in production
Common AI agents operate on brief, underspecified instructions. These work for simple demonstrations but are brittle in production — ambiguity leads to failure. The contractor model establishes a rigorous, formalized relationship built on clearly defined and mutually agreed-upon terms, much like a legal service agreement. This supports building trust required to deploy sophisticated agentic AI in mission-critical domains where reliability and accountability are paramount.
At a Glance

WHAT

Agentic systems operate in complex, dynamic environments where performance can degrade over time. Their probabilistic nature means traditional software testing is insufficient for ensuring reliability.

WHY

A standardized evaluation framework provides systematic performance assessment — defining metrics, analyzing trajectories, using LLM-as-a-Judge, and establishing feedback loops for continuous improvement.

WHEN

Use when deploying agents in live production environments, when comparing agent versions, in regulated domains requiring compliance audits, or when agents may drift due to changing data or environment.

RULE OF THUMB

Match evaluation rigor to stakes: exact trajectory matching for high-stakes scenarios; semantic similarity and LLM-as-Judge for nuanced outputs; automated metrics for scalable continuous monitoring.

Knowledge Quiz
Key Takeaways
  • Evaluating intelligent agents goes beyond traditional tests to continuously measure effectiveness, efficiency, and adherence to requirements in real-world environments.
  • Basic evaluation assesses response accuracy, while real-world scenarios demand sophisticated metrics like latency monitoring and token usage tracking for LLM-powered agents.
  • Agent trajectories — the sequence of steps an agent takes — are crucial for evaluation, comparing actual actions against an ideal, ground-truth path to identify errors and inefficiencies.
  • The LLM-as-a-Judge approach enables nuanced, human-like evaluation of subjective qualities like helpfulness, coherence, and appropriateness that resist objective measurement.
  • Google ADK provides structured evaluation methods: individual test files for unit testing and comprehensive evalset files for integration testing, with three execution paths (web UI, pytest, CLI).
  • Practical applications include performance tracking in live systems, A/B testing for improvements, compliance audits, and detecting drift or anomalies in behavior.
  • For reliable deployment in complex, high-stakes tasks, the paradigm is shifting from simple prompts to formal "contracts" that precisely define verifiable deliverables and execution scope.
  • Multi-agent system evaluation requires assessing cooperation, plan adherence, agent selection, and scalability — not just individual agent performance.
