"智能体画布"的比喻

构建智能代理就像在技术画布上创作复杂的艺术品。画布代表你的基础设施——LangChain、CrewAI 或 Google ADK。模式就是你的调色板和笔触。两者共同将简单的被动模型转变为主动的、目标导向的实体，能够进行复杂的推理和行动。

类别一：核心执行与任务分解

提示链（Prompt Chaining）、路由（Routing）、并行化（Parallelization）和规划（Planning）构成了智能体行动能力的基石。提示链将问题分解为线性步骤；路由添加条件逻辑；并行化实现子任务并发执行；规划将智能体从执行者提升为战略家。

类别二：与外部环境的交互

工具使用（Tool Use/函数调用）和知识检索（RAG）将智能体扎根于真实世界的数据中。没有这些模式，智能体只是一个孤立的信息孤岛；有了它们，智能体就能查询数据库、调用 API、搜索网络并使用实时信息。

类别三：状态、学习与自我改进

记忆管理（Memory Management）、反思（Reflection）、自我修正（Self-Correction）和学习适应（Learning & Adaptation）赋予智能体连续性和成长能力。一个无法记忆或改进的智能体永远被困在其初始能力的天花板之下。

类别四：协作与通信

多智能体协作（Multi-Agent Collaboration）、智能体间通信 A2A 和模型上下文协议（MCP）使专业智能体能够协同工作。当复杂问题分配给专家团队而不是单一通才时，原本棘手的挑战便迎刃而解。

智能体 AI 的挑战
为什么我们需要结构化的模式？以下是当智能体缺乏架构规范时会出现的核心失败模式。点击每张卡片展开详情。

不确定性

LLM 输出在每次运行时各不相同

与传统软件不同，使用 LLM 的智能体对相同输入会产生不同输出。这使得测试、调试和行为保证从根本上变得更加困难，需要全新的评估策略。

上下文丢失

智能体在长时间运行的任务中迷失目标

没有记忆管理，一个运行多步骤的智能体会逐渐失去上下文，导致行动相互矛盾，甚至完全忘记用户最初的意图。

幻觉现象

模型自信地生成不实信息

没有接地（RAG）和反思，智能体会自信地生成虚假信息。在金融或医疗等高风险领域，这是完全不可接受的。反思和知识检索模式直接解决了这一问题。

自主失控

智能体在缺乏适当监督的情况下行动

随着智能体获得工具访问权和执行真实世界行动的能力，不受约束的自主性变得危险。人机协同（Human-in-the-Loop）和防护栏（Guardrails）模式的存在就是为了提供关键检查点和安全边界。

协调复杂性

缺乏结构的多智能体系统会退化

当多个智能体在没有明确通信协议的情况下协作时，输出会变得混乱。A2A 和 MCP 模式标准化了智能体之间交换上下文、目标和结果的方式。

为什么模式是答案

智能体设计模式是经过实战检验的蓝图，能系统性地解决这些失败模式。它们不能消除构建智能体的复杂性——它们提供了经过验证的结构来管理这种复杂性，正如软件设计模式管理传统系统的复杂性一样。

模式作为解决方案框架
21 种模式不是独立选项的菜单——它们构成一个相互关联的框架。点击每个节点了解它们之间的关系。
1
基础层：执行模式
提示链 · 路由 · 并行化 · 规划
每个智能体系统都从这里开始。这些模式决定了工作如何被分解、排序和分派。在智能体能有效反思、协作或使用工具之前，它必须能够可靠地执行结构化工作流。规划模式尤其强大——它将智能体从被动的执行者提升为主动的战略家，能够为高层次目标制定多步骤方案。
2
接地层：世界交互
工具使用 · 知识检索（RAG）· MCP
孤立运行的 LLM 受限于其训练数据。接地模式将智能体连接到真实世界——API、数据库、搜索引擎和外部知识库。RAG 尤其能将智能体从静态知识的神谕转变为动态的研究者，可随需访问当前的权威信息。
3
智能层：状态与学习
记忆管理 · 反思 · 自我修正 · 学习与适应
这些模式赋予智能体连续性和改进能力。记忆使多轮对话连贯一致，支持长时间任务跟踪。反思和自我修正构建了质量反馈环，智能体可以迭代地批判自己的输出。学习与适应使行为能够从经验中进化——将静态智能体变为随时间不断提升能力的智能体。
4
协作层：多智能体系统
多智能体协作 · A2A · 人机协同 · 防护栏
最高阶的模式使智能体系统能够安全地协同工作。多智能体协作将复杂问题分配给由协调者统筹的专家智能体。人机协同在关键决策点插入人类判断。防护栏/安全模式确保系统在自主性增强时仍保持在可接受的边界内。
核心论点与综合
本书的中心论点：LLM 的原始力量需要架构规范才能成为可靠的、有目的的智能。

一种新语言的语法

智能体设计的原则是一种新创造语言的语法——它允许我们不仅指令机器做什么，而且指令它如何存在。真正的技艺不在于掌握单一模式，而在于理解它们的相互作用——构建一个规划、工具使用、反思和协作和谐运作的系统。

从提示到系统的旅程

本书记录的旅程逐步推进：单个提示 → 提示链 → 使用工具的智能体 → 自我修正的智能体 → 具有记忆的智能体 → 多智能体系统 → 安全、受监控、可投入生产的智能体平台。每一步都增加了一层由特定模式锚定的能力。

组合是核心洞见

单一模式不足以应对复杂的现实任务。AI 研究助理需要：规划（结构）+ 工具使用（接地）+ 多智能体协作（专业化）+ 反思（质量）+ 记忆管理（连贯性）。这种组合将各个能力转变为一个系统，其力量在质量上远超各部分之和。

模式是起点，不是教条

这 21 种模式代表了当前的最佳实践——一个坚实的基础，可在此基础上构建、实验和创新。该领域发展迅速。随着我们发现新的失败模式并开发新的解决方案，新模式将不断涌现。本书是对核心原则的邀请式反思，而不是最终规范。

实际应用：模式组合
本书通过一个具体示例——自主 AI 研究助理——来说明模式组合。探索以下每个组成部分。

自主 AI 研究助理

查询："分析量子计算对网络安全格局的影响。"这需要五种模式协同工作：

  • 规划：将查询分解为有序步骤的结构化研究计划
  • 工具使用：为每个步骤调用 Google 搜索、ArXiv API 和专业数据库
  • 多智能体协作："研究员"智能体收集原始数据；"写作者"智能体综合起草报告
  • 反思："评论家"智能体审查草稿，检查逻辑不一致和事实缺口
  • 记忆管理：在整个工作流中维护计划状态、收集的数据、草稿和反馈的状态

软件开发智能体

分配了一份 bug 报告，智能体必须修复、测试并提交代码补丁：

  • 规划：创建调试计划——重现、隔离、修复、测试
  • 工具使用：读取代码库、运行测试、访问文档 API
  • 反思：对照原始 bug 规格审查提议的修复方案
  • 人机协同：在提交前将补丁呈交人工审批
  • 异常处理：优雅地处理测试失败和回滚场景

客服智能体

处理具有完整上下文的复杂多轮客户咨询：

  • 记忆管理：记住完整的对话历史和客户档案
  • 路由：将账单问题、技术问题和投诉转给专业子智能体
  • 知识检索（RAG）：查询产品知识库以获取准确答案
  • 人机协同：对敏感或复杂情况升级到人工客服
  • 目标设定与监控：根据客户满意度指标跟踪解决情况

金融分析智能体

对公司或行业生成高精度研究报告：

  • 工具使用：调用金融数据 API 获取实时市场数据和 SEC 文件
  • 并行化：同时分析多家公司或数据流
  • 自我修正：验证数值计算并交叉核对数据来源
  • 防护栏/安全模式：防止生成未经授权的金融建议
  • 人机协同：在任何面向客户的输出前需要分析师签字确认

所有组合智能体系统共有的架构骨架——将多种模式联系在一起的编排器模式。

编排器架构

在任何复杂的智能体系统中，编排器智能体协调专业智能体的工作。这是使组合成为可能的结构性模式。

# 概念性编排器模式（源自第 7 章和结论） class OrchestratorAgent: def run(self, user_query: str) -> str: # 1. 规划：将目标分解为步骤 plan = self.planner_agent.create_plan(user_query) # 2. 执行：分派给专业智能体 results = [] for step in plan.steps: # 路由：选择合适的专家 agent = self.router.select_agent(step) # 工具使用：每个智能体使用其工具 result = agent.execute(step, context=results) results.append(result) # 3. 综合：合并输出 draft = self.writer_agent.synthesize(results, plan) # 4. 反思：批评并完善 critique = self.critic_agent.review(draft) if critique.has_issues: draft = self.writer_agent.revise(draft, critique) # 5. 人机协同：最终审批 return self.hitl.request_approval(draft)
未来发展方向
将模式组合为复杂系统只是软件开发新篇章的开始。以下几个新兴趋势将定义下一代智能系统。点击每层展开。
更强的自主性与推理能力
从"人在环路中"到"人监控环路"
未来的智能体将能够驾驭模糊性、进行抽象和因果推理，甚至展现常识。这需要与神经符号方法更紧密地结合，将 LLM 的模式匹配能力与经典 AI 的逻辑严谨性融合。我们将看到从人在环路（智能体作为副驾驶）向人监控环路的转变——智能体被信任执行复杂的长时间任务，只在目标完成或发生关键异常时才向人类汇报。
智能体生态系统与标准化
智能体即服务的开放市场
未来将出现开放市场，开发者可以在其中部署、发现和编排专业智能体群。为此，MCP 和 A2A 背后的原则将变得至关重要——推动行业范围内的标准，规范智能体、工具和模型之间不仅交换数据，还交换上下文、目标和能力。
安全性、对齐与鲁棒性
针对更自主智能体的新安全模式
随着智能体变得更加自主和互联，安全和对齐的核心问题变得更加关键。如何确保智能体的学习不会导致其偏离原始目的？如何构建能抵御对抗性攻击的系统？这需要一套新的"安全模式"和严格的工程规范，专注于测试、验证和伦理对齐。
智能体驱动的经济
高度自主的智能体作为经济参与者
未来的假设预测，智能体将作为独立经济实体——管理整个电商业务、自主开展科学研究，或作为专业服务团队的成员。2024 年智能体 AI 市场估值为 52 亿美元，预计到 2034 年将达到 2000 亿美元，这表明这不是遥远的前景，而是即将到来的现实。

最终核心要点
从本书中带走的最重要原则。
  • 智能体设计模式是可重用的架构蓝图——不是僵化的规则——能系统性地解决自主系统中已知的失败模式。掌握它们将使你从提示工程师跃升为系统架构师。
  • 21 种模式分为四个层次：执行、接地、智能和协作。每一层都建立在前一层之上。如果执行模式出现问题，协作模式就无法有效运作。
  • 组合是核心洞见。真实的智能体系统需要多种模式协同工作。AI 研究助理需要规划 + 工具使用 + 多智能体 + 反思 + 记忆才能在生产质量上可靠运行。
  • 画布比喻是实践性的，而非装饰性的。你选择的框架（LangChain、CrewAI、Google ADK）是你的画布。模式是你的笔触。不同的画布适合不同的用例；模式适用于所有画布。
  • 安全和人类监督不是可选的附加功能。人机协同和防护栏模式必须从一开始就设计进去，尤其是当智能体获得具有真实世界后果的工具访问权时。
  • 该领域正从"人在环路中"转向"人监控环路"。构建今天的系统，使其能够在信任建立和安全验证后优雅地提升自主性级别。
  • 这些模式是起点，不是最终答案。最重要的技能是能够识别何时适用现有模式，将其适应你的上下文，并与其他模式深思熟虑地组合。画布在你面前——现在是时候开始构建了。
