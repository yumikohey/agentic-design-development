本书简介

《智能体设计模式》是一本构建智能系统的实践指南，由 Antonio Gulli 撰写，Springer 出版。本书涵盖从基础的提示链到高级的多智能体协作的21个核心设计模式，每章均包含实际代码示例，使用 LangChain、LangGraph、CrewAI、Google ADK 等主流框架。全书版税将捐献给救助儿童会（Save the Children）。

作者 Antonio Gulli

Google 首席技术官办公室成员，在人工智能、机器学习和深度学习领域拥有丰富经验。专注于智能体系统的实践应用，致力于将复杂的AI技术以清晰、可操作的方式传授给开发者和研究人员。本书是其将理论与实践相结合的深度探索成果。

献辞

本书献给 Bruno、Leonardo、Lorenzo 和 Aurora——"这本书关于如何构建智能工具，但它致力于深切希望你们这一代人能用智慧和慈悲来引导它们。如果我们学会使用这些强大的技术服务人类并推动进步，未来对你们和我们所有人来说将无比光明。"

什么是 AI 智能体？

AI 智能体是能够感知环境、做出决策并采取行动以实现目标的自主系统。与传统AI不同，智能体具有持久性目标追踪、工具调用、与外部系统交互和多步推理的能力。本书系统地梳理了构建这类系统所需的核心设计模式。

缺乏系统性框架

智能体系统的设计缺乏成熟的模式库

不同于传统软件开发有成熟的设计模式（Gang of Four），AI智能体系统的构建缺乏系统性的模式指导，开发者往往需要从零开始摸索，导致重复造轮子和常见错误。

理论与实践脱节

学术研究与工程实现之间存在巨大鸿沟

大量学术论文描述了智能体的理论方法，但缺乏可直接应用的代码示例和工程实践指导。开发者需要将抽象概念转化为可运行的系统，这一过程充满挑战。

框架碎片化

LangChain、AutoGen、CrewAI等框架各自为政

智能体框架生态碎片化，每个框架都有独特的API和设计哲学。开发者需要理解多个框架的差异，并在项目中做出合理选择。本书通过对比帮助读者掌握核心概念而非特定API。

安全与可靠性缺失

生产级智能体系统的安全要求被低估

将智能体系统部署到生产环境需要处理安全、错误恢复、人机协作、内容过滤等复杂问题。许多开发者在原型阶段忽视了这些需求，导致系统难以可靠地大规模部署。

本书的系统性方法
本书通过四个递进层次系统地组织智能体设计知识：
一
第一部分：基础模式（103页）
第1-7章
涵盖构建智能体系统的核心基础：提示链、路由、并行化、反思、工具使用、规划和多智能体协作。这些是所有高级智能体系统的构建基石。每章包含完整代码示例，使用LangChain、LangGraph和Google ADK实现。
二
第二部分：中间层模式（61页）
第8-11章
深入探讨记忆管理、学习与适应、模型上下文协议（MCP）和目标设定与监控。这些模式使智能体能够在长时间交互中保持上下文、从经验中学习，并与外部系统标准化集成。
三
第三部分：可靠性模式（34页）
第12-14章
聚焦于生产级系统的可靠性：异常处理与恢复、人机协作（HITL）和知识检索（RAG）。这些模式确保智能体系统在真实环境中能够优雅地处理失败、保持人类监督并利用外部知识。
四
第四部分：高级模式（114页）
第15-21章
涵盖最复杂的智能体模式：智能体间通信、资源感知优化、推理技术、护栏与安全、评估与监控、优先级排序和探索与发现。这些模式用于构建真正自主、可扩展的生产级智能体系统。
核心主题

智能体设计模式的本质

类似于软件工程中的 Gang of Four 设计模式，智能体设计模式提供了解决常见问题的经验证方法。每个模式都是从真实实践中提炼的最佳实践，可在不同项目和框架中重用，帮助开发者避免常见陷阱并加速开发周期。

全书结构：21章 + 7附录

  • 第1-7章：提示链、路由、并行化、反思、工具使用、规划、多智能体协作
  • 第8-11章：记忆管理、学习适应、MCP协议、目标设定监控
  • 第12-14章：异常处理、人机协作、知识检索RAG
  • 第15-21章：智能体间通信、资源优化、推理技术、护栏安全、评估监控、优先级排序、探索发现
  • 附录A-G：高级提示、交互模式、框架概览、AgentSpace、CLI代理、推理引擎、编码代理

如何使用本书

本书适合三类读者：入门读者——按章节顺序阅读，从基础模式开始构建理解；有经验的开发者——可直接跳到感兴趣的具体模式章节；架构师——关注各模式之间的关系和组合方式。所有代码示例均可在配套仓库中获取。

代码示例风格
全书代码示例使用主流智能体框架。以提示链为例：
from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser from langchain_google_genai import ChatGoogleGenerativeAI # 提示链示例——市场分析工作流 llm = ChatGoogleGenerativeAI(model="gemini-2.0-flash") # 步骤1：总结报告 summarize_prompt = ChatPromptTemplate.from_template( """总结以下市场研究报告的关键发现：{report}""" ) # 步骤2：识别趋势 trends_prompt = ChatPromptTemplate.from_template( """基于以下总结，识别前三个新兴趋势：{summary}""" ) # 构建顺序链 chain = ( summarize_prompt | llm | StrOutputParser() | {"summary": lambda x: x} | trends_prompt | llm | StrOutputParser() ) result = chain.invoke({"report": market_report_text})

知识测验
核心要点
  • 本书提供了第一个系统性的智能体设计模式库，类似于软件工程的 Gang of Four 模式，填补了AI工程实践的重要空白。
  • 21个核心模式按四个递进层次组织：基础（第1-7章）→ 中间层（第8-11章）→ 可靠性（第12-14章）→ 高级（第15-21章）。
  • 每章均包含基于 LangChain、LangGraph、CrewAI 和 Google ADK 的可运行代码示例，实现了理论与实践的紧密结合。
  • 作者将全书版税捐献给救助儿童会，体现了对AI技术负责任发展和社会影响的深刻承诺。
  • 本书强调"设计模式"的可复用性——掌握这些模式后，读者可以在不同框架和项目中灵活应用，而不是依赖特定API的使用方法。
  • 从简单的提示链到复杂的多智能体探索系统，本书形成了完整的知识体系，覆盖了从原型到生产部署的全生命周期。
  • 生产级智能体系统的可靠性、安全性和监控（第12-19章）与功能实现同等重要，是本书的核心关注点之一。
  • AI智能体技术正在快速演变，本书提供的模式框架有助于读者在这一快速变化的领域建立持久的认知基础。
