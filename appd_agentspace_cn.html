使用 AgentSpace 构建智能体

AgentSpace 是 Google 的企业级 AI 平台，整合了统一搜索、知识图谱与无代码智能体创建功能——让组织无需深厚的编程专业知识，即可部署自主 AI 智能体。

什么是 AgentSpace？

AgentSpace 是一个旨在推动"智能体驱动企业"的平台，通过将人工智能融入日常工作流程来实现这一目标。其核心是提供跨越企业整个数字资产的统一搜索能力——包括文档、电子邮件和数据库——利用 Google 的 Gemini 模型来理解和综合来自不同来源的信息。

超越简单聊天机器人

该平台支持创建和部署专业化 AI 智能体，能够执行复杂任务并自动化流程。这些智能体不仅仅是聊天机器人——它们可以推理、规划并自主执行多步骤操作。例如，一个智能体可以研究某个主题、编写带有引用的报告，甚至生成音频摘要。

企业知识图谱

AgentSpace 构建企业知识图谱，绘制人员、文档和数据之间的关系。这使 AI 能够理解上下文并提供更相关、更个性化的结果。该平台还包含一个名为 Agent Designer 的无代码界面，无需深厚的技术专业知识即可创建自定义智能体。

通过 A2A 协议实现多智能体协作

AgentSpace 支持多智能体系统，不同的 AI 智能体可以通过名为 Agent2Agent（A2A）协议的开放协议进行通信和协作。这种互操作性支持更复杂的编排工作流。安全性是基础组件，具有基于角色的访问控制和数据加密功能，以保护敏感的企业信息。

企业 AI 面临的困境
在 AgentSpace 这类平台出现之前，企业在大规模部署 AI 智能体时面临重大障碍。点击各卡片了解具体挑战：

信息碎片化

数据分散在各系统孤岛中

企业数据分布在电子邮件、文档、数据库和 SaaS 工具中，没有统一的访问层。AI 模型无法跨这些来源进行连贯搜索，严重限制了自动化工作流的质量。

技术门槛高

需要深厚的工程专业知识

构建 AI 智能体历来需要大量编程知识。大多数业务用户和领域专家被排除在外，无法创建他们所需的智能体，减缓了 AI 在企业中的普及速度。

缺乏上下文感知

智能体不了解组织知识

通用 AI 模型对组织内具体的人员、项目、关系或历史一无所知。没有知识图谱，输出内容只能是通用回答，而非具有组织针对性的精确答案。

智能体孤立运行

智能体之间无法协作

各 AI 工具孤立运行。没有智能体间通信的标准协议，需要多个专业智能体协作的复杂多步骤任务就无法编排实现。

安全合规顾虑

企业敏感数据面临暴露风险

将 AI 系统连接到敏感企业数据引发了访问控制、数据泄露和合规方面的担忧。没有内置的基于角色的权限管理，广泛部署存在较大风险。

AgentSpace 的解决之道
AgentSpace 通过分层架构来应对企业 AI 挑战。点击每个步骤了解其工作原理：
1
通过 Google Cloud 控制台访问
入口：AI 应用程序
导航至 Google Cloud 控制台并选择"AI 应用程序"以访问 AgentSpace。您可以选择创建 Agentspace 应用（企业搜索+助手）或自定义搜索应用。Agentspace 选项由 Gemini 驱动，员工可以在海量公司数据中轻松找到答案、自动化内容创作并执行与连接应用相关的任务。
2
连接各类服务
集成：Google + 第三方平台
将您的智能体连接到各种服务。Google 原生来源包括 Calendar 和 Gmail；第三方来源包括 Workday、Jira、Outlook 和 ServiceNow。这些连接使您的智能体能够实时访问组织已在使用的工具，从而执行创建工单、起草电子邮件或预订假期等操作。
3
配置提示词
提示词库或自定义提示词
从 Google 预组装提示词库中选择（例如 goog_deep_research、goog_create_jira_ticket、goog_generate_code），或创建自己的自定义提示词。自定义提示词包含名称、显示名、标题、描述、提示类型、用户查询文本和激活行为。这定义了您的智能体如何解释和响应用户请求。
4
启用高级功能
知识图谱、分析统计、Web 界面
配置高级功能：启用 Google Cloud 知识图谱以用外部数据源扩展搜索结果；启用私有知识图谱，利用内部组织数据提供上下文准确的查询注释（更新后最长需要 24 小时重新生成）。添加连接的数据存储、启用面向 Web 的界面，并设置分析功能以监控使用情况。
5
部署并开始对话
实时智能体聊天界面
完成配置后，AgentSpace 聊天界面即可访问。用户可以搜索数据、提问并通过以"Google Agentspace"为品牌的简洁对话式界面与智能体互动。智能体可以在回答中引用多个来源，用户也可以直接在聊天输入框中附加来源。
核心概念
AgentSpace 建立在几个基础概念之上，使其与更简单的 AI 工具区别开来。点击各层探索：
企业知识图谱
绘制人员、文档和数据之间的关系
知识图谱是赋予 AgentSpace 上下文感知能力的智能层。它绘制组织数字资产之间的关系——谁认识谁、哪些文档与哪些项目相关、哪些数据属于哪个业务单元。这使 AI 能够提供精确的、个性化的结果，而非通用的回答。
Agent Designer（无代码界面）
无需编程专业知识即可构建智能体
Agent Designer 是 AgentSpace 用于创建自定义智能体的无代码界面。它抽象掉了底层的技术复杂性，让领域专家和业务用户通过选择服务、编写提示词和设置行为来配置智能体——无需任何代码。这使整个组织的 AI 智能体创建实现了民主化。
Agent2Agent（A2A）协议
多智能体通信的开放协议
A2A 协议是一个开放标准，使不同的 AI 智能体能够进行通信、协调和协作。这种互操作性对于编排复杂的企业工作流至关重要——需要多个专业智能体（例如研究智能体、写作智能体、调度智能体）相互传递任务并共享结果。
连接的数据存储
存储和检索组织的专有数据
AgentSpace 允许连接包含专有公司数据的自定义数据存储。这意味着智能体可以在内部文档、数据库和知识库上搜索和推理——这些内容在任何公共训练数据集中都不存在，使智能体真正具备组织专属知识。
基于角色的安全控制
默认启用访问控制和数据加密
安全性是 AgentSpace 的基础。基于角色的访问控制确保智能体只呈现用户有权查看的信息。数据加密保护静态和传输中的敏感企业信息。这使得将 AI 智能体连接到敏感的人力资源、财务或客户数据成为可行之举。

编程方式构建
虽然 AgentSpace 提供了无代码 UI，开发者也可以使用 Google Cloud API 以编程方式进行交互。以下是使用 Discovery Engine API 创建和配置 AgentSpace 智能体的方法：

前置条件

  • 已启用 AgentSpace 的 Google Cloud 项目
  • 项目已开启结算
  • 已启用 Discovery Engine API
  • 已配置应用默认凭证

安装所需 SDK：

pip install google-cloud-discoveryengine

创建 AgentSpace 数据存储：

from google.cloud import discoveryengine_v1 as discoveryengine # 初始化 DataStoreService 客户端 client = discoveryengine.DataStoreServiceClient() # 配置数据存储 data_store = discoveryengine.DataStore( display_name="企业知识库", industry_vertical=discoveryengine.IndustryVertical.GENERIC, content_config=discoveryengine.DataStore.ContentConfig.CONTENT_REQUIRED, ) # 在项目中创建数据存储 parent = f"projects/{PROJECT_ID}/locations/global/collections/default_collection" operation = client.create_data_store( parent=parent, data_store=data_store, data_store_id="my-enterprise-kb", ) result = operation.result() print(f"已创建数据存储：{result.name}")

创建 AgentSpace 智能体应用：

# 创建 AgentSpace 应用 engine_client = discoveryengine.EngineServiceClient() engine = discoveryengine.Engine( display_name="我的企业智能体", solution_type=discoveryengine.SolutionType.SOLUTION_TYPE_CHAT, chat_engine_config=discoveryengine.Engine.ChatEngineConfig( agent_creation_config=discoveryengine.Engine.ChatEngineConfig.AgentCreationConfig( business="我的组织", default_language_code="zh", time_zone="Asia/Shanghai", ) ), data_store_ids=["my-enterprise-kb"], ) operation = engine_client.create_engine( parent=parent, engine=engine, engine_id="my-enterprise-agent", ) result = operation.result() print(f"已创建智能体引擎：{result.name}")

UI 与 API 的权衡

无代码 Agent Designer UI 适合快速原型开发和赋能业务用户。API 方式更适合 CI/CD 流水线、基础设施即代码设置，以及对多个智能体进行程序化管理。两种方式最终都能产生相同的 AgentSpace 部署结果。

深入解析：架构与 A2A 协议
深入了解 AgentSpace 如何在底层编排复杂工作流，揭示它为何不仅仅是一个聊天机器人平台。

AgentSpace 架构层次

AgentSpace 在三个层次上运行：(1) 展示层 — 聊天 UI 和 API 端点；(2) 编排层 — Agent Designer 逻辑、提示词管理和 A2A 路由；(3) 知识层 — 企业知识图谱、连接的数据存储和 Gemini 推理。每一层都可以独立配置。

A2A 协议的实际运作

A2A 协议允许"管理者"智能体将子任务委托给专业化的"工作者"智能体。例如，研究智能体可以将文档检索委托给搜索智能体，将摘要编写委托给写作智能体。该协议定义了智能体如何公告其能力、接收和确认任务，以及如何返回结构化结果——实现了真正的智能体协作，而非简单的链式调用。

提示词库深度解析

Google 的预构建提示词库包含经过企业级验证的提示词，如 goog_deep_research（搜索增强型研究）、goog_explain_technical_documentation（技术文档解释）、goog_generate_code（代码生成）和 goog_help_me_analyze（数据分析与可视化）。这些提示词经过生产环境测试，遵循 Google 的安全和质量指南——在自定义之前是很好的起点。

私有知识图谱 vs. Google Cloud 知识图谱

AgentSpace 提供两种知识图谱模式。Google Cloud 知识图谱通过整合来自内部和外部数据源的丰富面板来增强搜索结果；私有知识图谱利用内部组织数据提供上下文精准的查询注释（更新后最长需 24 小时重新生成数据）。大多数企业会结合使用两者。

一览概要

问题（WHAT）

企业难以部署 AI 智能体，原因在于数据碎片化、技术门槛高、缺乏组织上下文感知，以及智能体孤立运行——无法将 AI 嵌入真实的业务工作流中。

方案（WHY）

AgentSpace 通过知识图谱统一企业数据，提供无代码 Agent Designer，集成 Google 和第三方服务，并使用 A2A 协议实现多智能体协作——一切均具备企业级安全保障。

关键机制

私有知识图谱绘制组织关系；提示词库提供预验证的智能体行为；A2A 协议支持编排多智能体工作流；基于角色的访问控制强制执行数据治理。

适用时机

当您需要企业级 AI 搜索、希望无需深度工程即可部署智能体、需要与 Google Workspace 或企业 SaaS 工具集成，或需要在 Google Cloud 上进行合规、安全的 AI 部署时，使用 AgentSpace。

核心洞察

AgentSpace 抽象了 AI 智能体部署的底层技术复杂性，使得无需深厚编程专业知识即可构建专业化的多智能体系统。其主要目标是将自动化的分析和操作能力直接嵌入工作流程，从而提高流程效率，并在企业规模上增强数据驱动的决策能力。

知识测验
测试您对 AgentSpace 核心概念的理解。
核心要点
  • AgentSpace 是 Google 的企业级 AI 平台，由 Gemini 模型驱动，提供统一搜索、知识图谱构建和无代码智能体创建功能。
  • 企业知识图谱绘制人员、文档和数据之间的关系，赋予智能体所需的组织上下文，实现精准、个性化的响应。
  • Agent Designer 是一个无代码界面，让业务用户无需编程即可通过选择服务、选择或创建提示词、配置行为来创建自定义智能体。
  • Agent2Agent（A2A）协议是一个开放标准，使不同 AI 智能体能够通信和协作，使复杂的编排多智能体工作流成为可能。
  • AgentSpace 与广泛的生态系统集成，包括 Google Calendar、Gmail、Workday、Jira、Outlook 和 ServiceNow——使智能体能够在企业系统中采取真实行动。
  • 安全性是基础：基于角色的访问控制确保智能体只呈现用户有权查看的信息，使连接敏感企业数据变得安全可行。
  • 如需实践学习，Google Cloud Skills Boost 上的"使用 Agentspace 构建生成式 AI 智能体"实验室提供了结构化的技能培训环境。

总结

AgentSpace 为在组织现有数字基础设施中开发和部署 AI 智能体提供了功能完备的框架。通过将复杂的后端流程——自主推理和企业知识图谱映射——与用于智能体构建的图形用户界面相连接，它使无需深厚编程专业知识即可构建专业化的多智能体系统成为可能。最终目标是将自动化的智能能力直接嵌入工作流程，大幅提升效率并实现规模化数据驱动决策。
