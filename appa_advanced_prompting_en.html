What is Prompt Engineering?

Prompting is the primary interface for interacting with language models — the process of crafting inputs to guide the model toward a desired output. This involves structuring requests, providing relevant context, specifying output format, and demonstrating expected response types. Well-designed prompts maximize the potential of language models, producing accurate, relevant, and creative responses.

The Engineering Mindset

Prompt engineering is a disciplined practice, not a simple act of asking questions. Its central purpose is to transform general-purpose language models into specialized, reliable, and highly capable tools for specific tasks. The journey begins with non-negotiable core principles like clarity, conciseness, and iterative experimentation — the bedrock of effective AI communication.

From Basics to Advanced

This appendix covers the full spectrum: from zero-shot and few-shot basics, through structural techniques like system prompts and delimiters, to reasoning patterns like Chain of Thought and Tree of Thoughts, and finally to action-oriented patterns like ReAct and RAG that power agentic systems.

Foundation for Agentic Systems

Agentic patterns — the architectural structures for building intelligent systems — define how agents plan, utilize tools, manage memory, and collaborate. The efficacy of these systems is contingent upon their ability to interact meaningfully with language models. Mastering prompting is therefore the definitive skill that elevates a language model from a text generator into a sophisticated cognitive engine.

What Goes Wrong with Poor Prompts
Poorly designed prompts lead to ambiguous, irrelevant, or erroneous outputs. Click each card to explore the failure modes that advanced techniques solve:

Vague Instructions

Model interprets ambiguously

Without clarity and specificity, language models may produce multiple valid but unintended interpretations. "Summarize this" with no format guidance yields unpredictable output lengths, styles, and focus areas.

No Reasoning Trace

Model skips complex logic

For multi-step math or logical deduction, a model asked for a direct answer often shortcuts reasoning and produces incorrect results. Chain of Thought solves this by requiring intermediate steps.

Unstructured Output

Cannot be parsed downstream

When output format is not specified, the model returns free-form prose that cannot be reliably parsed by the next step in an automated pipeline. This breaks agentic workflows that depend on machine-readable data.

Knowledge Cutoff Gaps

Outdated or hallucinated facts

Models limited to their training data produce hallucinated facts for time-sensitive queries. Without RAG or tool use prompting, an agent has no mechanism to access current, accurate external information.

Context Blindness

Stateless per-call behavior

Without context engineering, each model call is stateless. Agents that need to maintain goals, recall history, or integrate tool outputs across a session fail because the model lacks the operational picture required for coherent multi-step reasoning.

The Core Problem

The quality of a model's output depends more on the richness of the provided context and the structure of the prompt than on the model's architecture alone. Even advanced models underperform with a limited or poorly constructed view of their operational environment. Prompt engineering is the solution.

The Prompting Framework
Effective prompting is built in layers — from foundational principles, through structural techniques, to advanced reasoning and action patterns. Click each layer to explore:
1
Core Principles
Clarity · Conciseness · Iteration
The foundation: instructions must be unambiguous and precise. Use action verbs (Summarize, Extract, Classify) rather than vague phrasing. Prefer positive instructions over constraints. Treat every prompt as a first draft — iterate, test, and document.

Key insight: What is confusing to the user is likely confusing to the model. Simple, direct phrasing consistently outperforms elaborate instructions.
2
Structural Techniques
System Prompts · Role · Delimiters · Format
Structure prompts using sections for different information types. System prompts set context and behavior. Role prompting assigns expertise personas. Delimiters (XML tags, backticks) visually separate instructions from data. Requesting structured output (JSON, XML) ensures machine-readable results essential for pipelines.

Key insight: Structure reduces ambiguity and gives the model a clear map of what each piece of text represents.
3
Reasoning Techniques
CoT · Self-Consistency · ToT · Step-Back
For complex tasks, prompt the model to externalize its reasoning. Chain of Thought ("think step by step") dramatically improves multi-step logic. Self-Consistency generates multiple reasoning paths and selects the most consistent answer. Tree of Thoughts explores branching reasoning for exploratory problems. Step-Back prompting primes general principles before specifics.

Key insight: Generating intermediate steps keeps the model on track and enables interpretability of its reasoning.
4
Action & Context Techniques
ReAct · Tool Use · RAG · Context Engineering
Enable agents to interact with external systems. ReAct interleaves Thought → Action → Observation loops. Tool Use / Function Calling prompts the model to output structured JSON invocations for external APIs. RAG retrieves relevant documents before generation. Context Engineering dynamically builds the full operational picture — system prompts, retrieved docs, tool outputs, and implicit session state.

Key insight: These techniques are what grant an agent its ability to perceive and act upon its environment beyond its training data.
Core Concepts in Detail
Key techniques that power advanced prompting — from shot-based learning to reasoning patterns and structured output.

Shot-Based Prompting

Zero-Shot: No examples — relies entirely on pre-training. Best for tasks the model has seen extensively (simple Q&A, basic summarization). Quickest to write.

One-Shot: One input-output example. Guides the model on a specific format or style uncommon in training data.

Few-Shot: 3–5 examples demonstrating the pattern. Most effective for classification, structured extraction, and domain-specific style. Quality and diversity of examples are critical. Evolving into "many-shot" as context windows grow.

Chain of Thought (CoT)

Prompting the model to generate intermediate reasoning steps before the final answer. "Let's think step by step" triggers internal reasoning traces that dramatically improve accuracy on logic and math tasks.

Zero-Shot CoT: Simply append "Let's think step by step."
Few-Shot CoT: Provide examples with full reasoning traces shown.

Best practice: set temperature=0 for deterministic tasks; place the final answer after the reasoning steps.

ReAct Pattern

ReAct (Reason + Act) interleaves reasoning with tool actions in a loop:

1. Thought: Model explains current understanding and plan
2. Action: Model outputs a tool call (e.g., Search, Calculator)
3. Observation: System executes tool and returns result
4. Loop continues until Final Answer

This pattern is what gives agents the ability to gather information dynamically and react to real-world tool outputs.

Example: Structured output schema for tool calling

{ "tool_code": "get_current_weather", "tool_name": "get_current_weather", "parameters": { "city": "London" } }

Retrieval Augmented Generation (RAG)

RAG enhances models by giving them access to external, up-to-date, or domain-specific information during generation. The system retrieves relevant documents from a knowledge base, injects them as context, and the model generates a response grounded in that retrieved knowledge — mitigating hallucination and enabling access to post-training information.

Use Cases by Technique
Each prompting technique shines in specific scenarios. Explore seven practical application contexts:

Code Prompting

LLMs are powerful coding assistants when prompted correctly. Specify the language, version, and context clearly.

1
Generate: "Write a Python function that takes a list of numbers and returns the average." — provide language and functionality.
2
Explain: "Explain the following JavaScript code snippet line by line: [insert code]."
3
Debug: "The following Python code gives a 'NameError'. What is wrong and how can I fix it? [insert code and traceback]."
4
Translate: "Translate the following Java code to C++: [insert code]."

Complex Reasoning (CoT / ToT)

Chain of Thought is ideal for multi-step math, logical deduction, and planning tasks where a direct answer shortcuts correct reasoning.

1
Append "Let's think step by step" or provide CoT examples for math word problems, scheduling, causal analysis.
2
Use Self-Consistency for high-stakes reasoning: run the prompt 5x at high temperature and take the majority answer.
3
Use Step-Back prompting for creative tasks: first ask for general principles, then apply them to the specific problem.
4
Use Tree of Thoughts for exploratory problems requiring backtracking and evaluation of multiple candidate solutions.

Structured Data Extraction

Requesting structured output (JSON, XML) is essential for automated pipelines where the LLM output feeds into downstream processing.

1
Specify the exact schema: "Extract name, address, and phone_number as a JSON object."
2
Use Pydantic models to validate and parse the JSON output — combine parsing and validation in one step.
3
Provide a schema example in the prompt to demonstrate the exact structure expected.
4
Handle ValidationError gracefully — retry with a corrective prompt if the output does not match the schema.

RAG Pipelines

RAG is the key pattern for agentic systems that need to work with dynamic or proprietary information.

1
User asks: "What are the new features in Python library X version 3.2?"
2
System retrieves relevant documentation snippets from a vector knowledge base.
3
Retrieved text is injected as context: "Based on the following documentation: [retrieved text], explain the new features."
4
Model generates a grounded response — no hallucination risk on outdated knowledge.

Tool Use / Function Calling

Agents use tool descriptions in the prompt to decide when and how to invoke external capabilities.

1
Describe available tools: name, purpose, and parameters in the system prompt.
2
Model outputs a structured JSON tool invocation instead of a text answer.
3
Agentic system executes the tool and returns the Observation.
4
Model integrates the result in its next Thought and produces a final answer.

Content Generation

Role prompting and few-shot examples significantly improve content generation quality for specific domains, styles, and audiences.

1
Assign a role: "Act as a seasoned travel blogger. Write a short, engaging paragraph about the best hidden gem in Rome."
2
Use Persona Pattern to tailor output: "The target audience is a high school student. Explain simply using analogies."
3
Use Factored Cognition: decompose a research paper into outline → section drafts → final revision prompts.

Meta-Prompting (APE)

Use the LLM itself to critique and improve prompts — Automatic Prompt Engineering (APE) turns AI into a collaborative prompt engineering partner.

1
Provide the model with an existing prompt and its failing outputs.
2
"Analyze this prompt and suggest ways to improve it to consistently extract the main topic and key entities."
3
Model identifies ambiguities, suggests delimiters, format clarifications, or better examples.
4
Apply suggestions, test on goldset, measure improvement — iterate programmatically with DSPy-style optimization.
Code Examples
Practical implementations of key prompting techniques — structured output extraction with Pydantic and a full ReAct trace.

Structured Output with Pydantic Validation

from pydantic import BaseModel, EmailStr, Field, ValidationError from typing import List, Optional from datetime import date # Define the schema the LLM must conform to class User(BaseModel): name: str = Field(..., description="The full name of the user.") email: str = Field(..., description="The user's email address.") date_of_birth: Optional[date] = Field(None) interests: List[str] = Field(default_factory=list) # Hypothetical LLM JSON output llm_output_json = """ { "name": "Alice Wonderland", "email": "alice.w@example.com", "date_of_birth": "1995-07-21", "interests": ["NLP", "Python", "Gardening"] } """ # Parse and validate in one step try: user_object = User.model_validate_json(llm_output_json) print(f"Name: {user_object.name}") print(f"Email: {user_object.email}") except ValidationError as e: print("Failed to validate JSON from LLM.") print(e)

ReAct Trace — Illustrative Example

# User query: "What is the capital of France and its current population?" # --- ReAct Loop --- Thought: The user needs two pieces of information. I'll use search to confirm the capital, then find its population. Action: Search({"query": "capital of France"}) Observation: Paris Thought: Capital is Paris. Now I need the population. Action: Search({"query": "current population of Paris 2024"}) Observation: ~2.1 million (city proper), ~12 million (metro area) Thought: I have both pieces of information. Ready to answer. Final Answer: The capital of France is Paris. Its city population is approximately 2.1 million people.
Deep Dive: Context Engineering
Context engineering is the evolution beyond static prompting — dynamically constructing the full operational picture an agent needs. Click each layer to expand:
Layer 1: System Prompts
Foundational operational parameters
System prompts set the AI's designated purpose, response style, and knowledge domain. They persist across a session and define the agent's identity. Example: "You are a technical writer; your tone must be formal and precise." They can be automatically optimized using tools like Google's Vertex AI Prompt Optimizer — systematically improving them against user-defined metrics.
Layer 2: Retrieved Documents
Information actively fetched from knowledge bases
RAG pipelines pull relevant documents, specifications, or previous conversation summaries into the prompt at runtime. This grounds the model in current, factual reality — preventing it from relying on potentially outdated training data. The quality and relevance of retrieved documents directly determine response accuracy.
Layer 3: Tool Outputs
Results from external API calls and computations
When an agent executes a tool (calculator, calendar API, database query), the result is injected back into context as an Observation. This creates a feedback loop where the model's next Thought is grounded in real, computed data rather than a probabilistic estimate. Tool outputs are the "senses" of an agentic system.
Layer 4: Implicit Data
User identity, session history, environmental state
Critical information such as user identity, interaction history, and session state. Including implicit context requires careful governance for privacy and ethical data management — especially in enterprise, healthcare, and finance contexts. This layer enables personalization and continuity across sessions.
Layer 5: Advanced Optimization
Automated prompt refinement with DSPy and APE
DSPy treats prompts as programmatic modules optimized against a goldset of input-output pairs. A Bayesian optimizer systematically refines both few-shot examples and instructional text. Google Gems pre-configure persistent contextual parameters for specialized, repeatable tasks. Meta-prompting uses the LLM itself to critique and suggest improvements — accelerating iteration cycles dramatically.
At a Glance
Quick reference for all major prompting techniques covered in this appendix.

ZERO / ONE / FEW-SHOT

Provide 0, 1, or 3–5 examples of input→output pairs. More examples improve format and style adherence. Many-shot scales with context window.

CHAIN OF THOUGHT (CoT)

"Think step by step" unlocks intermediate reasoning traces. Use Self-Consistency (majority vote across runs) for high-accuracy needs.

TREE OF THOUGHTS (ToT)

Explore multiple reasoning branches concurrently. Best for creative or exploratory tasks requiring backtracking and evaluation.

ReAct

Interleave Thought → Action → Observation loops. Enables real-time tool use and dynamic information gathering during reasoning.

RAG

Retrieve relevant documents from a knowledge base before generation. Grounds responses in current facts, eliminates hallucination on dynamic data.

CONTEXT ENGINEERING

Dynamically build the agent's full operational picture: system prompt + retrieved docs + tool outputs + session state. Quality of context = quality of output.

Key Takeaways
The essential lessons from Appendix A on Advanced Prompting Techniques.
  • Prompting is a disciplined engineering practice — clarity, conciseness, and iterative experimentation are non-negotiable foundations. Action verbs and positive instructions consistently outperform vague or constraint-heavy prompts.
  • Shot-based techniques (zero-, one-, few-shot) provide the model with behavioral examples. Quality and diversity of examples matter more than quantity; few-shot is evolving into many-shot as context windows grow.
  • Chain of Thought prompting ("think step by step") dramatically improves accuracy on reasoning, math, and logic tasks by externalizing intermediate steps — and improves interpretability as a bonus.
  • Structured output (JSON, XML validated with Pydantic) is not a convenience — it is an absolute necessity for robust agentic pipelines where one step's output feeds the next. It prevents catastrophic parsing failures.
  • ReAct interleaves reasoning and action in a Thought→Action→Observation loop, enabling agents to use tools, gather live information, and refine their approach dynamically rather than relying solely on training data.
  • RAG and Context Engineering together form the agent's senses: RAG retrieves external knowledge, while context engineering builds a comprehensive operational picture from system prompts, tool outputs, and session state.
  • Automatic Prompt Engineering (APE) and meta-prompting leverage LLMs themselves to critique and optimize prompts — accelerating iteration and enabling scalable, data-driven prompt tuning with frameworks like DSPy.
