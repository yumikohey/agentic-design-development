What is AgentSpace?

AgentSpace is a platform designed to facilitate an "agent-driven enterprise" by integrating artificial intelligence into daily workflows. At its core, it provides unified search across an organization's entire digital footprint — documents, emails, and databases — using Google's Gemini models to comprehend and synthesize information from varied sources.

Beyond Simple Chatbots

The platform enables the creation and deployment of specialized AI agents that can perform complex tasks and automate processes. These agents are not merely chatbots — they can reason, plan, and execute multi-step actions autonomously. For instance, an agent could research a topic, compile a report with citations, and even generate an audio summary.

Enterprise Knowledge Graph

AgentSpace constructs an enterprise knowledge graph, mapping the relationships between people, documents, and data. This allows the AI to understand context and deliver more relevant and personalized results. The platform also includes a no-code interface called Agent Designer for creating custom agents without requiring deep technical expertise.

Multi-Agent Collaboration via A2A

AgentSpace supports a multi-agent system where different AI agents can communicate and collaborate through an open protocol known as the Agent2Agent (A2A) Protocol. This interoperability allows for more complex and orchestrated workflows. Security is foundational, with role-based access controls and data encryption to protect sensitive enterprise information.

The Enterprise AI Problem
Before platforms like AgentSpace, enterprises faced significant barriers to deploying AI agents at scale. Click each card to explore the challenges:

Fragmented Information

Data siloed across systems

Enterprise data lives in email, docs, databases, and SaaS tools with no unified access layer. AI models could not search across these sources coherently, limiting the quality of automated workflows.

High Technical Barrier

Requires deep engineering expertise

Building AI agents historically required significant programming knowledge. Most business users and domain experts were excluded from creating the agents they needed, slowing adoption.

No Context Awareness

Agents lack organizational knowledge

Generic AI models have no understanding of an organization's specific people, projects, relationships, or history. Without a knowledge graph, outputs were generic rather than contextually precise.

Agent Isolation

Agents cannot collaborate

Individual AI tools operated in isolation. Without a standard protocol for agent-to-agent communication, complex multi-step tasks requiring multiple specialized agents were impossible to orchestrate.

Security Concerns

Enterprise data exposure risks

Connecting AI systems to sensitive enterprise data raised concerns about access control, data leakage, and compliance. Without role-based permissions baked in, widespread deployment was risky.

AgentSpace as the Solution
AgentSpace addresses enterprise AI challenges through a layered architecture. Click each step to explore how it works:
1
Access via Google Cloud Console
Entry Point: AI Applications
Navigate to the Google Cloud Console and select "AI Applications" to access AgentSpace. You'll choose between creating an Agentspace app (enterprise search + assistant) or a Custom search app. The Agentspace option is powered by Gemini and allows employees to find answers across vast company data and execute tasks with connected apps.
2
Connect Services
Integrations: Google + Third-Party
Connect your agent to a wide range of services. Google-native sources include Calendar and Gmail. Third-party sources include Workday, Jira, Outlook, and ServiceNow. These connections give your agent real-time access to the tools your organization already uses, enabling it to take actions like creating tickets, drafting emails, or booking time off.
3
Configure Prompts
Prompt Gallery or Custom Prompts
Choose from Google's gallery of pre-assembled prompts (e.g., goog_deep_research, goog_create_jira_ticket, goog_generate_code) or create your own custom prompt. Custom prompts include a Name, Display Name, Title, Description, Prompt Type, User Query text, and Activation Behavior. This defines how your agent interprets and responds to user requests.
4
Enable Advanced Features
Knowledge Graph, Analytics, Web UI
Configure advanced capabilities: enable the Google Cloud Knowledge Graph to expand search results with external data sources; enable a Private Knowledge Graph that leverages internal organizational data for contextually accurate query annotations (may take up to 24 hours to regenerate). Add connected datastores, enable the web-facing interface, and set up analytics to monitor usage.
5
Deploy and Chat
Live Agent Interface
Upon completion, the AgentSpace chat interface becomes accessible. Users can search data, ask questions, and interact with the agent through a clean conversational UI branded as "Google Agentspace." The agent can reference multiple sources in responses and users can attach sources directly in the chat input.
Core Concepts
AgentSpace is built on several foundational concepts that distinguish it from simpler AI tools. Click each layer to explore:
Enterprise Knowledge Graph
Maps relationships between people, documents, and data
The knowledge graph is the intelligence layer that gives AgentSpace its contextual awareness. It maps relationships across an organization's digital assets — who knows whom, which documents relate to which projects, which data belongs to which business unit. This allows the AI to deliver precise, personalized results rather than generic answers.
Agent Designer (No-Code Interface)
Build agents without programming expertise
Agent Designer is AgentSpace's no-code interface for creating custom agents. It abstracts the underlying technical complexity, letting domain experts and business users configure agents by selecting services, writing prompts, and setting behavior — without any code. This democratizes AI agent creation across the organization.
Agent2Agent (A2A) Protocol
Open protocol for multi-agent communication
The A2A Protocol is an open standard that enables different AI agents to communicate, coordinate, and collaborate. This interoperability is critical for orchestrating complex enterprise workflows where multiple specialized agents (e.g., a research agent, a writing agent, a scheduling agent) need to hand off tasks and share results.
Connected Data Stores
Store and retrieve your organization's proprietary data
AgentSpace allows you to connect custom data stores containing proprietary company data. This means agents can search and reason over internal documents, databases, and knowledge bases that are not available in any public training dataset — giving them access to truly organization-specific information.
Role-Based Security
Access controls and data encryption by default
Security is foundational in AgentSpace. Role-based access controls ensure agents only surface information that a user is authorized to see. Data encryption protects sensitive enterprise information at rest and in transit. This makes it viable to connect AI agents to sensitive HR, financial, or customer data.

Building Programmatically
While AgentSpace offers a no-code UI, developers can also interact with it programmatically using the Google Cloud APIs. Here is how to create and configure an AgentSpace agent using the Discovery Engine API:

Prerequisites

  • Google Cloud project with AgentSpace enabled
  • Billing enabled on the project
  • Discovery Engine API enabled
  • Application Default Credentials configured

Install the required SDK:

pip install google-cloud-discoveryengine

Create an AgentSpace data store and connect it to a corpus:

from google.cloud import discoveryengine_v1 as discoveryengine # Initialize the DataStoreService client client = discoveryengine.DataStoreServiceClient() # Configure the data store data_store = discoveryengine.DataStore( display_name="Enterprise Knowledge Base", industry_vertical=discoveryengine.IndustryVertical.GENERIC, content_config=discoveryengine.DataStore.ContentConfig.CONTENT_REQUIRED, ) # Create the data store in your project parent = f"projects/{PROJECT_ID}/locations/global/collections/default_collection" operation = client.create_data_store( parent=parent, data_store=data_store, data_store_id="my-enterprise-kb", ) # Wait for the operation to complete result = operation.result() print(f"Created data store: {result.name}")

Configure an AgentSpace app with the data store:

# Create an AgentSpace application engine_client = discoveryengine.EngineServiceClient() engine = discoveryengine.Engine( display_name="My Enterprise Agent", solution_type=discoveryengine.SolutionType.SOLUTION_TYPE_CHAT, chat_engine_config=discoveryengine.Engine.ChatEngineConfig( agent_creation_config=discoveryengine.Engine.ChatEngineConfig.AgentCreationConfig( business="My Organization", default_language_code="en", time_zone="America/Los_Angeles", ) ), data_store_ids=["my-enterprise-kb"], ) operation = engine_client.create_engine( parent=parent, engine=engine, engine_id="my-enterprise-agent", ) result = operation.result() print(f"Created engine: {result.name}")

UI vs. API Trade-offs

The no-code Agent Designer UI is ideal for rapid prototyping and business user empowerment. The API approach is better for CI/CD pipelines, infrastructure-as-code setups, and programmatic management of many agents. Both approaches result in the same underlying AgentSpace deployment.

Deep Dive: Architecture & A2A
Understanding how AgentSpace orchestrates complex workflows under the hood reveals why it is more than just a chatbot platform.

The AgentSpace Architecture Stack

AgentSpace operates on three layers: (1) the Presentation Layer — the chat UI and API endpoints; (2) the Orchestration Layer — the Agent Designer logic, prompt management, and A2A routing; (3) the Knowledge Layer — the enterprise knowledge graph, connected data stores, and Gemini inference. Each layer is independently configurable.

Agent2Agent (A2A) Protocol in Practice

The A2A Protocol allows a "manager" agent to delegate sub-tasks to specialized "worker" agents. For example, a research agent might delegate document retrieval to a search agent, and summarization to a writing agent. The protocol defines how agents advertise their capabilities, how they receive and acknowledge tasks, and how they return structured results — enabling genuine agent collaboration rather than simple chaining.

Prompt Gallery Deep Dive

Google's pre-built prompt gallery includes enterprise-hardened prompts like goog_deep_research (uses search_check_spark icon), goog_explain_technical_documentation (menu_book_spark), goog_generate_code (data_object), and goog_help_me_analyze (text_analysis). These are production-tested and follow Google's safety and quality guidelines — a strong starting point before customizing.

Private vs. Public Knowledge Graph

AgentSpace offers two knowledge graph modes. The Google Cloud Knowledge Graph expands search with external data sources, broadening scope with additional insights. The Private Knowledge Graph leverages your internal organizational data for contextually accurate query annotations — highly personalized but requires up to 24 hours to regenerate after updates. Most enterprises use both in combination.

At a Glance

PROBLEM

Enterprises struggle to deploy AI agents due to fragmented data, high technical barriers, lack of organizational context awareness, and agent isolation — preventing AI from being embedded in real workflows.

SOLUTION

AgentSpace unifies enterprise data via a knowledge graph, provides a no-code Agent Designer, integrates with Google and third-party services, and uses the A2A Protocol for multi-agent collaboration — all with enterprise-grade security.

KEY MECHANISM

The Private Knowledge Graph maps organizational relationships. Prompt gallery provides pre-hardened agent behaviors. A2A Protocol enables orchestrated multi-agent workflows. Role-based access controls enforce data governance.

WHEN TO USE

Use AgentSpace when you need enterprise-wide AI search, want to deploy agents without deep engineering, require integration with Google Workspace or enterprise SaaS tools, or need a compliant, secure AI deployment on Google Cloud.

Key Insight

AgentSpace abstracts the underlying technical complexity of AI agent deployment, enabling the construction of specialized multi-agent systems without requiring deep programming expertise. The primary objective is to embed automated analytical and operational capabilities directly into workflows, thereby increasing process efficiency and enhancing data-driven analysis.

Key Takeaways
  • AgentSpace is Google's enterprise AI platform that provides unified search, knowledge graph construction, and no-code agent creation powered by Gemini models.
  • The Enterprise Knowledge Graph maps relationships between people, documents, and data — giving agents the organizational context needed for precise, personalized responses.
  • Agent Designer is a no-code interface that lets business users create custom agents by selecting services, choosing or creating prompts, and configuring behavior without programming.
  • The Agent2Agent (A2A) Protocol is an open standard enabling different AI agents to communicate and collaborate, making complex orchestrated multi-agent workflows possible.
  • AgentSpace integrates with a broad ecosystem including Google Calendar, Gmail, Workday, Jira, Outlook, and ServiceNow — enabling agents to take real actions across enterprise systems.
  • Security is foundational: role-based access controls ensure agents only surface information a user is authorized to see, making it safe to connect agents to sensitive enterprise data.
  • For hands-on learning, the "Build a Gen AI Agent with Agentspace" lab on Google Cloud Skills Boost provides a structured environment for skill acquisition.

Summary

AgentSpace provides a functional framework for developing and deploying AI agents within an organization's existing digital infrastructure. By linking complex backend processes — autonomous reasoning and enterprise knowledge graph mapping — to a graphical user interface for agent construction, it enables the creation of specialized multi-agent systems without deep programming expertise. The goal is to embed automated, intelligent capabilities directly into workflows, increasing efficiency and enabling data-driven decision-making at scale.
