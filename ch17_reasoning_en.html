What are Reasoning Techniques?

This chapter delves into advanced reasoning methodologies for intelligent agents, focusing on multi-step logical inferences and problem-solving. These techniques go beyond simple sequential operations, making the agent's internal reasoning explicit. This allows agents to break down problems, consider intermediate steps, and reach more robust and accurate conclusions.

Compute at Inference Time

A core principle among these advanced methods is the allocation of increased computational resources during inference. This means granting the agent, or the underlying LLM, more processing time or steps to process a query and generate a response. Rather than a quick, single pass, the agent can engage in iterative refinement, explore multiple solution paths, or utilize external tools. This extended processing time during inference often significantly enhances accuracy, coherence, and robustness — especially for complex problems requiring deeper analysis.

The Thought-Action-Observation Loop

At its core, an agent's "thinking" is facilitated by a powerful LLM generating a series of thoughts that guide subsequent actions. The process follows a thought-action-observation loop: (1) The agent generates a textual thought that breaks down the problem; (2) Based on the thought, the agent selects an action; (3) The agent receives feedback from its environment. This cycle repeats until the agent determines it has reached a final solution.

The Problem: Single-Pass Reasoning Fails
Without explicit reasoning techniques, LLMs produce shallow outputs for complex tasks. Click cards to explore failure modes:

Missing Intermediate Steps

Complex math and logic collapse without step-by-step breakdown

A model asked "If a store sells apples for $0.50 each and oranges for $0.75, and I buy 3 apples and 4 oranges, how much do I spend?" often gives the wrong answer when it tries to solve it in one step. Breaking it into sub-computations dramatically improves accuracy.

No Backtracking

Models commit to the first reasoning path without exploring alternatives

Standard CoT generates a single linear path. If the first reasoning direction is flawed, the model cannot backtrack and try another approach. Tree-of-Thought was designed specifically to address this limitation.

Static Knowledge Cutoff

Models cannot access current information without external tools

A purely generative model is frozen at its training cutoff. For questions about current events, live data, or real-time calculations, the model needs to act — query a database, call an API, or perform a web search. ReAct enables this interaction.

Opaque Decision-Making

Without visible reasoning, errors are hard to diagnose

When a model produces an incorrect answer in one step, there is no chain of reasoning to inspect. Explicit reasoning techniques like CoT make the model's thought process visible, enabling debugging and trust-building.

Imprecise Computation

LLMs hallucinate in arithmetic and symbolic tasks

Language models are not calculators. PALMs (Program-Aided Language Models) solve this by having the model generate code that performs the computation deterministically, then interpreting the result in natural language.

Single-Agent Bias

One model's reasoning trajectory may be biased or incomplete

A single AI producing an answer may have systematic biases from its training. Chain of Debates (CoD) and Graph of Debates (GoD) use multiple diverse models that argue, critique, and counterargue — reducing bias through collective intelligence.

The Solution: Structured Reasoning Frameworks
Reasoning techniques provide structured frameworks that make thinking explicit and robust. Click each step to explore:
1
Chain-of-Thought (CoT)
Make reasoning explicit via intermediate steps
CoT prompting guides the model to generate a sequence of intermediate reasoning steps before producing a final answer. Instead of "What is 2+2?" → "4", CoT produces: "I need to add 2 and 2. Two plus two equals four."

Implementation: Few-shot examples demonstrating step-by-step reasoning, or simply instructing the model to "think step by step."

Why it works: Forces the model to explicitly consider sub-problems, reducing the chance of skipping key logical transitions. Makes reasoning auditable.
2
Self-Correction / Self-Refinement
Iteratively review and improve generated content
The agent evaluates its own generated content and intermediate thought processes. This critical review identifies ambiguities, information gaps, or inaccuracies. The iterative cycle of reviewing and refining allows the agent to adjust its approach, improve response quality, and ensure accuracy before delivering a final output.

Example: A Self-Correction Agent reviews a draft social media post, identifies weaknesses (generic phrasing, weak call to action), proposes improvements, and rewrites: "Discover GreenTech Gadgets! Eco-friendly innovation. Go green! #EcoFriendly"
3
ReAct (Reasoning + Acting)
Interleave reasoning with tool use
ReAct integrates CoT prompting with the agent's ability to interact with external environments through tools. Unlike purely generative models, a ReAct agent reasons about which actions to take, acts by executing tool calls (database queries, calculations, web searches), observes the result, and incorporates this observation into subsequent reasoning.

Loop: Thought → Action → Observation → Thought → …

This iterative loop allows the agent to dynamically adapt its plan, correct errors, and achieve goals requiring multiple environment interactions.
4
Tree-of-Thought (ToT)
Explore multiple reasoning paths with backtracking
ToT builds upon CoT by allowing LLMs to explore multiple reasoning paths by branching into different intermediate steps, forming a tree structure. This supports complex problem-solving by enabling backtracking, self-correction, and exploration of alternative solutions.

How: At each step, the model generates multiple candidate next steps, evaluates them, and expands the most promising ones. Less promising branches are pruned.

Maintaining a tree of possibilities allows the model to evaluate various reasoning trajectories before finalizing an answer — particularly powerful for strategic planning and multi-step optimization.
Core Reasoning Techniques
Seven key techniques form the landscape of advanced agent reasoning. Click each to expand:
Chain-of-Thought (CoT)
Step-by-step explicit reasoning
CoT prompting significantly enhances LLMs' complex reasoning abilities by mimicking a step-by-step thought process. Instead of a direct answer, CoT prompts guide the model to generate a sequence of intermediate reasoning steps. This transforms a difficult single-step problem into a series of simpler steps, increasing transparency. CoT is implemented via few-shot examples or the simple instruction "think step by step." It has become a cornerstone technique for enabling advanced reasoning in contemporary LLMs.
Tree-of-Thought (ToT)
Multiple paths with backtracking
ToT extends CoT by branching into different intermediate steps, forming a tree structure. The model evaluates various reasoning trajectories before finalizing an answer. This iterative process enhances the model's ability to handle challenging tasks requiring strategic planning. It supports backtracking, self-correction, and exploration of alternative solutions — capabilities not available in linear CoT.
PALMs (Program-Aided Language Models)
LLMs + code execution for precise computation
PALMs integrate LLMs with symbolic reasoning by having the model generate and execute code (Python) as part of its problem-solving. They offload complex calculations, logical operations, and data manipulation to a deterministic programming environment. This hybrid approach uses traditional programming strengths where LLMs exhibit limitations in accuracy or consistency. When faced with symbolic challenges, the model produces code, executes it, and converts results into natural language.
RLVR (Reinforcement Learning with Verifiable Rewards)
Train models to reason through trial and error
RLVR is the training strategy enabling "reasoning models." By training on problems with known correct answers (math, code), the model learns to generate effective, long-form reasoning through trial and error. These models dedicate a variable amount of "thinking" time — producing a dynamic Chain-of-Thought that can be thousands of tokens long. The result: reasoning trajectories demonstrating advanced skills like planning, monitoring, and evaluation. This is how models like o1, o3, and DeepSeek-R1 were trained.
ReAct (Reasoning and Acting)
Interleave thought, action, and observation
ReAct integrates CoT with tool interaction. The agent reasons about which action to take, executes it (database query, API call, web search), observes the result, and incorporates the observation into the next thought. This Thought→Action→Observation loop enables dynamic plan adaptation, error correction, and goals requiring multiple environment interactions. It is crucial for agents that must not only reason but practically execute steps and interact with dynamic environments.
Chain of Debates (CoD)
Multiple models argue toward consensus
A formal AI framework proposed by Microsoft where multiple, diverse models collaborate and argue to solve a problem. This system operates like an AI council meeting: different models present initial ideas, critique each other's reasoning, and exchange counterarguments. The primary goal is to enhance accuracy, reduce bias, and improve overall quality by leveraging collective intelligence. It represents a shift from a solitary agent to a collaborative team of agents.
Scaling Inference Law
More compute at inference time = better results
This critical principle dictates that a model's performance predictably improves as the computational resources allocated during inference increase. Crucially, a smaller LLM given a more substantial "thinking budget" (via multiple candidates, iterative refinement, or extended CoT) can occasionally surpass a much larger model using simple generation. This challenges the "bigger is always better" paradigm and enables more economically viable AI deployments.

Hands-On Code
Concrete examples of reasoning techniques in practice.

Chain-of-Thought Prompt: Information Retrieval Agent

This 5-step CoT prompt instructs the agent how to think before answering:

# Chain-of-Thought system prompt COT_PROMPT = """ You are an Information Retrieval Agent. Your goal is to answer the user's question comprehensively and accurately by thinking step-by-step. Here's the process you must follow: 1. **Analyze the Query:** Understand the core subject and specific requirements. Identify key entities, keywords, and the type of information being sought. 2. **Formulate Search Queries:** Generate a list of precise search queries to retrieve relevant information. 3. **Simulate Information Retrieval:** For each search query, consider what information you expect to find. Think about potential ambiguities or missing pieces. 4. **Synthesize Information:** Based on your analysis, synthesize the gathered information into a coherent and complete answer. 5. **Review and Refine:** Critically evaluate your answer. Is it accurate? Is it comprehensive? Is it concise? """

PALMs: Agent with Code Executor (Google ADK)

Separate agents for search and code execution, orchestrated by a root agent:

from google.adk.tools import agent_tool from google.adk.agents import Agent from google.adk.tools import google_search from google.adk.code_executors import BuiltInCodeExecutor search_agent = Agent( model='gemini-2.0-flash', name='SearchAgent', instruction="You're a specialist in Google Search", tools=[google_search], ) coding_agent = Agent( model='gemini-2.0-flash', name='CodeAgent', instruction="You're a specialist in Code Execution", code_executor=[BuiltInCodeExecutor], ) root_agent = Agent( name="RootAgent", model="gemini-2.0-flash", description="Root Agent", tools=[ agent_tool.AgentTool(agent=search_agent), agent_tool.AgentTool(agent=coding_agent) ], )

DeepSearch with LangGraph: Iterative Research Loop

A ReAct-style agent graph that searches, reflects, and refines until context is sufficient:

from langgraph.graph import StateGraph, START, END # Build the agent graph builder = StateGraph(OverallState, config_schema=Configuration) # Define nodes: generate query → web search → reflection → finalize builder.add_node("generate_query", generate_query) builder.add_node("web_research", web_research) builder.add_node("reflection", reflection) builder.add_node("finalize_answer", finalize_answer) builder.add_edge(START, "generate_query") builder.add_conditional_edges( "generate_query", continue_to_web_research, ["web_research"] ) builder.add_edge("web_research", "reflection") # Reflection decides: more research needed or finalize? builder.add_conditional_edges( "reflection", evaluate_research, ["web_research", "finalize_answer"] ) builder.add_edge("finalize_answer", END) graph = builder.compile(name="pro-search-agent")

Chain of Debates (CoD): A formal AI framework by Microsoft where multiple diverse models collaborate and argue to solve a problem. Different models present initial ideas, critique each other's reasoning, and exchange counterarguments. The primary goal is to enhance accuracy, reduce bias, and improve quality via collective intelligence. Functions as an AI peer review — creating a transparent and trustworthy record of the reasoning process.

MASS: Multi-Agent System Search

MASS automates the design of multi-agent systems via a 3-stage optimization process: (1) Block-Level Prompt Optimization — locally optimize prompts for individual agent types before integration; (2) Workflow Topology Optimization — select and arrange agent interactions from a design space using influence-weighted search; (3) Workflow-Level Prompt Optimization — globally fine-tune prompts for the entire system after the best topology is found. Key finding: MAS optimized by MASS significantly outperform manually designed systems.

Scaling Inference Law in Practice

This law states performance predictably improves as inference-time compute increases. A smaller model given a larger "thinking budget" — via diverse beam search, self-consistency methods, or extended CoT — can occasionally surpass a much larger model using simple generation. This is fundamental to constructing efficient agentic systems: it provides a methodology for balancing model size, response latency, and operational cost.

At a Glance

WHAT

Advanced reasoning methodologies — CoT, ToT, ReAct, PALMs, RLVR, CoD, GoD — that go beyond simple sequential operations, making the agent's internal reasoning explicit for multi-step problem-solving.

WHY

LLMs using single-pass generation fail on complex tasks: missing intermediate steps, no backtracking, static knowledge cutoff, opaque decisions. Explicit reasoning techniques make thinking auditable and robust.

THOUGHT-ACTION-OBS

The core ReAct loop: agent generates a thought, selects an action, executes it via tools, observes the result, and loops. This enables dynamic, environment-responsive problem solving.

SCALING INFERENCE

More inference-time compute can beat a larger model using simple generation. A smaller model with extended thinking budget (CoT, multiple candidates) often outperforms a larger model on hard tasks.

Technique Selection Guide

  • Use CoT when you need to improve accuracy on multi-step reasoning tasks with no tool access
  • Use ToT when multiple solution paths exist and backtracking is needed (e.g., puzzles, strategy)
  • Use ReAct when the agent must interact with external environments (search, APIs, databases)
  • Use PALMs when precise arithmetic or symbolic computation is required
  • Use RLVR models (o1, o3, DeepSeek-R1) when the task is inherently hard and demands deep deliberation
  • Use CoD/GoD when consensus from multiple perspectives reduces bias risk

Key Takeaways
  1. Chain-of-Thought (CoT) is the cornerstone technique — it makes reasoning explicit by guiding the model to produce intermediate reasoning steps, dramatically improving accuracy on arithmetic, commonsense reasoning, and symbolic manipulation tasks.
  2. Tree-of-Thought (ToT) extends CoT by branching into multiple reasoning paths, enabling backtracking and exploration of alternatives — essential for strategic planning, puzzles, and complex optimization.
  3. Self-Correction / Self-Refinement builds quality control directly into the agent's content generation cycle — reviewing outputs against original requirements and iteratively improving them before delivery.
  4. ReAct (Reasoning and Acting) is fundamental for agents that must interact with dynamic environments — interleaving thought, action, and observation in a loop enables real-time adaptation and error correction.
  5. RLVR (Reinforcement Learning with Verifiable Rewards) is the training paradigm behind modern reasoning models (o1, o3, DeepSeek-R1) — enabling extended dynamic Chain-of-Thought with thousands of tokens, supporting self-correction and backtracking.
  6. The Scaling Inference Law challenges "bigger is always better" — a smaller model with a generous inference-time compute budget can outperform a larger model using simple generation, enabling more economically viable AI deployments.
