Quick Overview of Agentic Frameworks

A practical tour of the major frameworks for building LLM-powered applications — from simple linear chains to full multi-agent orchestration platforms.

Framework Landscape

The agentic framework ecosystem spans a wide spectrum — from foundational component libraries to opinionated multi-agent platforms. Choosing the right one depends on how much control vs. convenience your application requires.

LangChain — Linear Pipelines

LangChain's core strength is its Expression Language (LCEL), which lets you "pipe" components together into a chain. It is ideal for Directed Acyclic Graph (DAG) workflows where the process flows in one direction without loops.

LangGraph — Stateful Graphs

Built on top of LangChain, LangGraph lets you define workflows as graphs with nodes and edges. Its key advantage is supporting cycles — enabling loops, retries, and tool use until a task is complete.

Google ADK — Multi-Agent Orchestration

Google's Agent Development Kit provides pre-built architectural patterns for multi-agent interaction. It abstracts away graph construction, offering built-in agent types like SequentialAgent and ParallelAgent.

CrewAI — Role-Based Teams

CrewAI models agents as a team of specialists with distinct personas, roles, and goals. Developers define Agents, Tasks, and a Crew — CrewAI manages the collaborative interaction process.

Why Do We Need Specialized Frameworks?

Building reliable agentic applications from scratch involves solving hard recurring problems. Click each card to understand the challenge.

State Management

LLMs are stateless by default. Complex workflows need to pass context across multiple steps and agent boundaries.

Without explicit state objects, data must be manually threaded through every function call. Frameworks like LangGraph solve this with a typed state dictionary that persists across the entire workflow.

Control Flow Complexity

Agents need to loop, branch, retry, and make conditional decisions — impossible to express as a simple function chain.

DAG-based frameworks like LangChain cannot handle cycles. Graph-based frameworks like LangGraph introduce edges with conditional logic to route between nodes dynamically.

Multi-Agent Coordination

Systems with multiple specialized agents require orchestration logic to delegate, route, and aggregate results.

High-level platforms like Google ADK and CrewAI provide pre-built patterns — supervisor agents, sequential crews, hierarchical delegation — so developers don't need to build coordination from scratch.

Tool Integration Boilerplate

Connecting LLMs to external APIs, search tools, databases, and code executors involves repetitive scaffolding.

Frameworks provide standardized tool interfaces and function-calling integration, letting developers focus on defining what tools do rather than how to call them from an LLM.

Observability & Debugging

Multi-step LLM workflows are opaque — it's hard to trace why a particular path was taken or where errors occurred.

Frameworks like SuperAGI and production-oriented platforms include built-in monitoring, logging, and failure-handling mechanisms that catch common issues like infinite loops.

Core Concepts by Framework

Each framework has a distinct mental model. Understanding the key abstractions is essential before writing any code.

LangChain: LCEL and Chains

The core abstraction is a chain built with the LangChain Expression Language (LCEL). Components are composed using the pipe operator |. The workflow is a Directed Acyclic Graph — each step's output becomes the next step's input. State is generally stateless per run.

# LCEL chain: prompt | model | output_parser chain = prompt | model | output_parser result = chain.invoke({"topic": "machine learning"})

LangGraph: Nodes, Edges, and State

The core abstraction is a StateGraph. You define a typed state dictionary, then add nodes (Python functions) and edges (control flow). Edges can be conditional, enabling cycles. The state object is updated by each node and passed to the next.

CrewAI: Agents, Tasks, and Crews

Three building blocks: an Agent has a role, goal, and backstory. A Task is a discrete unit of work with a description and expected output. A Crew binds agents and tasks together and executes a Process (sequential or hierarchical).

Google ADK: LlmAgent and Tools

The core primitive is an LlmAgent configured with a model, name, instructions, and a list of tools. ADK handles the agent loop implicitly. For multi-agent systems, agents can be composed into SequentialAgent or ParallelAgent containers that manage flow automatically.

Framework Selection Guide

Choose the right framework for your use case. Each tab describes a scenario and the best framework choice.

Simple RAG Pipeline

Best framework: LangChain (LCEL)

1
User query arrives
2
Retriever fetches relevant documents from vector store
3
Prompt template formats context + query
4
LLM generates answer; output parser formats response

LangChain's LCEL is perfect here. The workflow is linear (A → B → C), has no loops, and maps exactly to a pipe chain: retriever | prompt | model | parser.

Tool-Using Agent with Retry Logic

Best framework: LangGraph

1
Agent receives task and generates a plan
2
Agent calls a tool (search, calculator, API)
3
Conditional edge: if result is unsatisfactory, loop back to step 2
4
When task is complete, exit loop and return final answer

LangGraph's cyclical graph handles the retry loop naturally. The state object tracks tool call history across iterations.

Research Report with Specialist Agents

Best framework: CrewAI or Google ADK

1
Researcher agent searches the web for information
2
Analyst agent synthesizes findings into structured data
3
Writer agent composes the final report
4
Editor agent reviews and refines the draft

CrewAI's role-based model is a natural fit. Each agent has a defined persona and task; the Crew runs them sequentially with context passing between tasks.

Plan-and-Execute Agent

Best framework: LangGraph

1
Planner node creates a multi-step plan for the goal
2
Executor node carries out the next step in the plan
3
Evaluator node checks if the step succeeded
4
Loop back to planner to update the plan based on results, or exit when done

LangGraph's explicit state passing allows the plan, completed steps, and observations to all live in the shared state object, updated across each loop.

Enterprise Software Integration

Best framework: Semantic Kernel

1
Existing .NET or Python codebase provides native functions as plugins
2
Semantic Kernel planner determines which plugins to invoke for a user goal
3
LLM acts as a reasoning engine orchestrating existing business logic
4
Results are returned through the standard application layer

Semantic Kernel's plugin architecture is built for integrating into existing enterprise codebases, particularly in .NET and Python environments.

Knowledge-Intensive RAG System

Best framework: LlamaIndex

1
Ingest large document corpus into a sophisticated index structure
2
Build custom retrieval pipelines with re-ranking and filtering
3
Query engine handles complex nested queries across multiple data sources
4
Agent uses retrieved context for evidence-backed responses

LlamaIndex excels at sophisticated data ingestion and retrieval pipelines. Its native tools for complex agentic control flow are less developed, but for RAG it is exceptional.

Code Examples

Concrete Python examples from the three primary frameworks covered in this appendix.

LangChain — Simple LCEL Chain

# A simple LCEL chain: conceptually demonstrates the pipe flow from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser prompt = ChatPromptTemplate.from_template( "Summarize the following text in one sentence:\n\n{text}" ) model = ChatOpenAI(model="gpt-4o") output_parser = StrOutputParser() # The pipe operator chains components together chain = prompt | model | output_parser result = chain.invoke({"text": "LangChain enables composable LLM applications..."}) print(result)

LangGraph — Parallel Fan-Out with Aggregator

from langgraph.graph import StateGraph, START, END from typing import TypedDict # Graph state class State(TypedDict): topic: str joke: str story: str poem: str combined_output: str # Node functions — each updates a slice of state def call_llm_1(state: State): msg = llm.invoke(f"Write a joke about {state['topic']}") return {"joke": msg.content} def call_llm_2(state: State): msg = llm.invoke(f"Write a story about {state['topic']}") return {"story": msg.content} def aggregator(state: State): combined = f"JOKE:\n{state['joke']}\n\nSTORY:\n{state['story']}" return {"combined_output": combined} # Build and compile the graph builder = StateGraph(State) builder.add_node("call_llm_1", call_llm_1) builder.add_node("call_llm_2", call_llm_2) builder.add_node("aggregator", aggregator) builder.add_edge(START, "call_llm_1") builder.add_edge(START, "call_llm_2") builder.add_edge("call_llm_1", "aggregator") builder.add_edge("call_llm_2", "aggregator") builder.add_edge("aggregator", END) workflow = builder.compile() state = workflow.invoke({"topic": "cats"}) print(state["combined_output"])

Google ADK — Search-Augmented Agent

from google.adk.agents import LlmAgent from google.adk.tools import google_search # Define an agent with a tool — ADK manages the agent loop search_agent = LlmAgent( model="gemini-2.0-flash-exp", name="research_agent", description="An agent that answers questions using web search.", instruction="""Answer the user's question using the google_search tool to find current, accurate information from the web.""", tools=[google_search], ) # The agent loop (observe → think → act) is handled implicitly response = search_agent.run("What are the latest developments in LangGraph?")
Framework Comparison Deep Dive

A structured comparison of the four primary frameworks across key dimensions. Click each layer to explore the analysis.

LangChain
Core abstraction: Chain (LCEL) · Workflow: Linear DAG · State: Stateless per run
LangChain operates at the most foundational level, offering components and standardized interfaces. Its LCEL pipe syntax is clean and readable. Best for: Simple RAG pipelines, summarization chains, data extraction from text. The limitation is the absence of cycles — you cannot loop back or retry based on results without LangGraph.
LangGraph
Core abstraction: Graph of Nodes · Workflow: Cyclical · State: Explicit persistent object
LangGraph extends LangChain with a stateful graph runtime. Developers explicitly define nodes (functions), edges (transitions), and conditional edges (branching logic). The TypedDict state is the single source of truth, updated by each node. This enables looping agents, plan-and-execute patterns, and human-in-the-loop workflows. Best for: Complex agents that reason, use tools, and retry.
Google ADK
Core abstraction: LlmAgent + Sub-agents · Workflow: Team-oriented · State: Implicitly managed
Google's ADK abstracts graph construction with pre-built agent types. SequentialAgent runs sub-agents in order; ParallelAgent runs them concurrently. A primary orchestrator agent delegates to specialists. State and session management are handled more implicitly, providing cohesion at the cost of granular control. Best for: Production multi-agent applications where you want a managed "fleet" of agents.
CrewAI
Core abstraction: Agent + Task + Crew · Workflow: Sequential or Hierarchical · State: Task context passing
CrewAI occupies a distinct position by modelling collaboration as a human team. Agents have rich personas (role, goal, backstory) that guide behavior. Tasks have explicit expected outputs. The developer designs a "team charter" rather than a state machine. CrewAI concentrates specifically on agent collaboration logic. Best for: Multi-agent systems that mirror human team workflows — research, analysis, and writing pipelines.
Other Frameworks: AutoGen, LlamaIndex, Haystack, MetaGPT, SuperAGI, Strands
Specialized tools for specific use cases
AutoGen: Conversation-driven multi-agent coordination. Agents interact through dialogue, enabling complex decomposition. Less predictable execution paths. LlamaIndex: Exceptional for sophisticated RAG and data-intensive applications. Haystack: Production-ready search systems with modular pipeline nodes. MetaGPT: SOP-driven agent collaboration mimicking a software company. SuperAGI: Full lifecycle management with monitoring and observability. Strands Agents: AWS lightweight SDK — model-agnostic, MCP-native, simple customizable agent loop.

Key Takeaways
  • LangChain's LCEL provides a clean pipe-based syntax for building linear, DAG-style LLM workflows. It is the right choice for predictable, sequential pipelines.
  • LangGraph extends LangChain with cyclical graphs and explicit state management. Use it whenever your agent needs to loop, retry, or make conditional decisions based on intermediate results.
  • The key LangChain vs. LangGraph decision: if your flow can be described as A → B → C without looping, use LangChain. If the agent needs to reason, use tools, and potentially try again, use LangGraph.
  • Google's ADK abstracts away low-level graph construction, providing pre-built agent types (SequentialAgent, ParallelAgent) for orchestrating teams of specialized agents.
  • CrewAI models multi-agent collaboration as a human team with defined roles, goals, and backstories. It focuses on the "who does what" rather than "how the graph is wired."
  • Specialized frameworks serve distinct niches: LlamaIndex for data-intensive RAG, Haystack for production search, MetaGPT for SOP-driven code generation, Strands for lightweight AWS-native agents.
  • The core trade-off across all frameworks is granular control (graph-based) vs. streamlined development (opinionated platforms). The right choice depends on whether your application requires a simple sequence, a dynamic reasoning loop, or a managed team of specialists.
  • The agentic framework ecosystem is rapidly evolving. The best framework is the one that matches the precise level of abstraction your application demands — not necessarily the most popular one.
